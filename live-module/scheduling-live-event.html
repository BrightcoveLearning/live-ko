--- title: 라이브 SEP 이벤트 예약 parent: Live Module ---

정적 진입점이 있는 라이브 이벤트의 자동 시작/중지

이 기능을 통해 사용자는 하나 이상의 라이브 작업 이벤트의 시작 및 중지 시간을 예약하고 관리 할 수 있습니다.

전제 조건

이벤트를 예약하려면 SEP (Static Entry Point) 유형의 라이브 이벤트를 만들어야합니다.

현재 중복 그룹의 스케줄링은 지원되지 않습니다.

권장 사항 및 모범 사례

최상의 결과를 얻으려면 아래 지침을 따르십시오.

  1. 대기 중인 6개의 활성화를 한 번에 예약할 수 있습니다. 더 필요한 경우 고객 지원에 문의하십시오.
  2. SEP가 활성화되면 기본 최대 대기 시간은 여전히 유효합니다 ( 라이브 API 참조: 자세한 내용은 정적 진입 점 ).
  3. 정적 진입점에 설명된 제한사항은 여전히 유효합니다.

제어실의 스케줄링 패널

라이브 계정이 자동 시작 / 중지에 대해 활성화 된 경우 미리보기 플레이어 아래의 제어실 오른쪽 상단 모서리에서 예약 패널을 사용할 수 있습니다.

자동 시작 / 중지를 사용하려면 아래 지침을 따르십시오.

  1. 라이브 모듈에서 정적 진입 점이있는 새 라이브 이벤트를 만들거나 이전에 만든 이벤트를 방문합니다.
  2. 제어실에서 Scheduling이라는 새 패널을 사용할 수 있습니다. + 일정추가를 클릭하여 새 자동 활성화 및 비활성화를 만듭니다.
  3. 시작 및 종료 날짜를 각각 원하는 활성화 및 비활성화로 수정합니다.
    여러 일정이 동시에 실행되는 것을 방지하지는 않지만 충돌하는 행은 경고와 함께 표시됩니다. 활성화 시간 이전에 해결되지 않으면 충돌하는 일정이 실패 할 수 있습니다. 경고를 방지하려면 행 중 하나를 삭제하거나 시간을 조정하면 됩니다.
  4. 두 번째 일정을생성하려면+ 일정 추가버튼을 다시 클릭합니다. 보류 중인 활성화 제한에 도달하면 + 일정 추가버튼이 자동으로 비활성화됩니다.
  5. 일정을 삭제하려면 상.
  • 스케줄 이벤트는 충돌하는 스케줄이있을 때 경고로 노란색으로 바뀝니다.
  • 어떤 이유로 든 스트리밍에 실패하면 예약 된 이벤트가 빨간색으로 바뀝니다.
  • 실패한 일정은 7 일 동안 표시됩니다.

예정된 활성화보기

  1. 예약된 활성화가 생성되면 이벤트관리 > 대기탭에서 확인할 수 있습니다. 다음 예정된시작및중지날짜는 다음제목 아래에 표시됩니다.
  2. NS 보여 주다선택 상자를 사용하면 예약된 이벤트만 표시할 수 있습니다.
  3. 이벤트가 시작되면라이브탭에 표시되고시작시간이 비활성화됩니다.
