사용자 지정 라이브 수집 프로필 필드

이 항목에서는 Live 용 수집 프로필에 사용되는 필드에 대한 참조를 제공합니다.

프로필 필드

부동산 유형 기술 기본
account_id 정수 비디오 클라우드계정 ID 없음
name 끈 프로필 이름-계정 내에서 고유해야합니다. 없음
display_name 끈 Studio Admin 페이지에 표시할 프로파일의 표시 이름은 다음과name같을 수 있지만 사용자가 프로필의 용도를 이해하는 데 도움이 되도록 설명적이어야 합니다. 없음
description 끈 프로필 설명 없음
model_version 정수 라이브 모듈에 프로필이 표시되도록1하려면 이 옵션을 설정합니다. 없음
renditions 정렬 변환 객체 배열 []
packages 정렬 DRM 패키지 배열-현재 DRM은 지원되지 않으므로 항상 빈 배열이어야합니다. []

변환 필드

아래 표는 라이브 수집 프로필을 업데이트하기 위해 요청 본문에서 사용할 수있는 필드를 보여줍니다.

부동산 유형 기술 기본
audio_bitrate 정수 오디오 비트 레이트 (kbps) (최소: 16, 최대: 1024) 일치 소스
audio_codec 끈 오디오 코덱- aac | mp3 형식 또는 프로필에 따라 결정됨 < - 대부분의aac경우 최고/td>
decoder_bitrate_cap 정수 kbps, 버퍼를 통해 디코더에 공급되는 최대 비트 전송률 없음
decoder_buffer_size 정수 kbps의 경우 디코더에 공급되는 버퍼의 크기bitrate_cap 없음
forced_keyframe_rate 번호 X 프레임마다 키 프레임을 적용하지만 장면 자르기 등을위한 추가 키 프레임은 계속 허용합니다. 없음
format 끈 비디오 형식 - ts , mp4 , flv 없음
h264_profile 끈 H.264에는 일반적으로 사용되는 세 개의 프로파일이 있습니다. Baseline (최저) Main , 및High .[1-5] baseline
hls_optimized_ts 부울 HLS에 최적화 된 시간 세그먼트-HLS 출력에만 적용 없음
keyframe_interval 정수 키 프레임 사이의 최대 프레임 수 250
keyframe_rate 흙손 초당 최대 키 프레임 keyframe_interval 250의
label 끈 이미지 변환의 이미지 유형 - poster또는thumbnail 없음
live_stream 부울 이것이 라이브 비디오 스트림인지 여부 없음
live_dvr_sliding_window_duration 정수 LiveDVR 전달에 사용할 수있는 스트림 시간 (초) 300
max_video_bitrate 정수 kbps, 123 = 123kbps-h264 만 없음
max_frame_rate 흙손 을 (를) 설정하는frame_rate것이 아니라 제한합니다. 60 FPS 지원에 의해 계정이 활성화되지 않은 경우 30으로 설정해야합니다. 없음
media_type 끈 video또는audio 없음
video_bitrate 정수 초당 단위, 123 = 123킬로바이트당 (최소: 64, 최대: 10000) 없음
video_codec 끈 예: h264, ts 형식 또는 프로필에 따라 결정
width 정수 프레임 너비 (픽셀) 없음
height 정수 프레임 높이 (픽셀) 없음
