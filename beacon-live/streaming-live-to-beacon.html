--- title: 실시간 이벤트를 Beacon parent: Live Module 앱으로 스트리밍합니다.

비콘 앱으로 라이브 이벤트 스트리밍

이 항목에서는 Beacon 앱으로 라이브 이벤트를 스트리밍하는 방법을 배웁니다.

개요

라이브 이벤트를 애플리케이션으로 스트리밍하려면 일련의 단계에 따라 Video Cloud Live 계정과 Brightcove Beacon을 연결해야합니다.

프로세스에 대한 상위 수준보기는 다음과 같습니다.

  1. 첫 번째 단계는 Brightcove Beacon에서 이벤트 및 재생 목록을 만드는 것입니다. 이 이벤트는 앱에서 동영상의 자리 표시 자로 사용할 수 있습니다.
  2. Video Cloud 계정으로 이동하고 Live에 액세스하여 이벤트를 만들고 식별자로 작동 할 작업 ID를 얻으십시오.
  3. Brightcove Beacon으로 돌아 가면 플랫폼의 이벤트 섹션 아래에 라이브 플랫폼에서 생성 된 작업 ID를 넣어 라이브 스트리밍을 연결할 수 있습니다.
  4. 연결을 기다리면 Beacon 앱에서 라이브 비디오 스트리밍이 시작됩니다.
  5. 마지막 섹션에서는 정적 진입 점으로 라이브에서 다른 유형의 이벤트를 만드는 단계를 볼 수 있습니다.

이 문서의 나머지 부분에서는 이러한 각 단계에 대한 세부 정보를 제공합니다.

Beacon에서 라이브 이벤트 만들기:

브라이트코브 비콘 플랫폼에서이벤트탭을 클릭하여 이벤트 설정을 시작합니다.

브라이트코브 Beacon Live에서 활성화된 계정만 탭에서재생 목록및이벤트를사용할 수 있습니다.섹션. 활성화되지 않은 경우 브라이트코브 지원에 문의하십시오.
  1. 새 이벤트추가를 클릭합니다 .
  2. 라이브 이벤트에이름및이벤트 시작 날짜를지정합니다.
  3. 텍스트 데이터탭에서 이벤트 정보를 입력합니다.
  4. 텍스트가 아닌 데이터탭에서 이벤트에 사용할 이미지 포스터를 선택합니다.

    세로, 가로, 썸네일, 와이드 배너 중에서 이벤트에 사용할 포스터 유형을 선택할 수 있습니다.

  5. 권한 및 예약탭에서가용성 세부 정보를 다음과같이 설정할 수 있습니다.
    • 권한 유형: 광고, 구독 또는 무료 중에서 권리 유형을 선택하십시오.
    • 시작 시간: 이벤트 시작 시간
    • 종료 시간: 이벤트 종료 시간인 이 시간은 앱 내에서 한도를 생성하기 위해 생성됩니다.
    • 최대 스트림: 앱에서 허용되는 최대 스트림 수입니다. 기본 숫자는 0 (영)입니다.
    • 디바이스 세부 정보: 여기에서 스트림을 게시 할 여러 장치를 추가 할 수 있습니다. (예: iOS, 안드로이드, 로쿠, 삼성 등)
    • 지리적 위치 세부 정보: 앱을 활성화하려는 국가 목록입니다. 국가를 제한하고 싶지 않은 경우 "세계"를 선택할 수 있습니다.
    • 거부된 위치: 콘텐츠에 대한 액세스가 거부 된 지역 목록입니다.
  6. 아래 버튼을 클릭합니다. 새 이벤트 만들기

재생 목록에 비콘 이벤트 추가

이벤트를 만들고 나면 상단 메뉴의재생 목록탭으로 이동합니다. 스트리밍하기 전에 이벤트를 재생 목록에 추가해야합니다.

  1. 새 재생 목록추가버튼을 클릭하고 새로 만듭니다.
    라이브 이벤트를 포함할 Brightcove Beacon에서 재생 목록을 만들어야 합니다. 비디오 클라우드의 재생 목록을사용하지 마십시오. 비디오 클라우드 재생 목록을 사용하면 Beacon Live 녹화를 덮어쓰거나 삭제할 수 있습니다.
  2. 재생 목록의콘텐츠 유형 탭에서이벤트탭을 선택합니다.
  3. 콘텐츠 모듈에서 이벤트를 찾고 [+) 버튼을 클릭하여 왼쪽 재생 목록에 추가합니다. 앱에 표시하려는 방식으로 이벤트 순서를 배치 할 수 있습니다.
  4. 재생 목록 설정 (시간, 장치, 위치 등)을 조정합니다. 이벤트와 마찬가지로.
  5. 재생 목록에 변경 사항저장버튼을 클릭합니다.
  6. 캐시를 지우려면 화면 오른쪽 상단의 도구 버튼을 클릭하면 모든 것이 앱에 올바르게 표시됩니다.

이 시점에서 이벤트는 Brightcove Beacon 앱에서 컨테이너로 생성되지만 실제로 라이브 비디오 스트리밍을 시작하려면 Video Cloud Live Platform에서 전송을 설정해야합니다.

Studio에서 라이브 이벤트 만들기

라이브 모듈에 액세스하려면 Video Cloud Studio에 로그인하고 탐색 헤더에서 라이브를 클릭하십시오.

  1. 이벤트 관리 페이지에서 이벤트생성을 누릅니다 . 라이브 이벤트 만들기 또는 라이브 채널 만들기 페이지가 열립니다. 이벤트와 채널을 생성 할 때의 속성은 동일합니다. 이 문서의 스크린 샷은 이벤트 생성을 다룹니다.
  2. 이벤트 세부 정보를 입력합니다.
    • 이벤트 명 -이벤트 이름 이 이름은 비디오가 비디오 자산 (최대 255 자)으로 저장된 경우 미디어 모듈에도 나타납니다.
    • 간단한 설명 -이벤트 설명 (최대 250 자)
    • 태그 -이벤트와 관련된 태그 비디오가 비디오 자산으로 저장되면 비디오에 태그가 할당됩니다 (최대 128 자 및 1200 태그).
    • 참조 ID -이벤트에 대한 참조 ID 고유해야합니다 (최대 150 자).
  3. 계정에 필요한 모든 사용자 정의 필드가 표시됩니다. 사용자 정의 필드에 값을 지정하십시오.
  4. 스트리밍을 소셜 미디어로허용됨으로 설정하면 이벤트를 소셜 미디어 플랫폼으로 스트리밍할 수 있습니다. 이 작업은 이벤트가 스트리밍을시작한 후또는 Social 모듈을 사용하여 수행할 수있습니다.
  5. 라이브 인제스트 프로파일을 선택합니다 . 드롭 다운 목록에는 모든 라이브 수집 프로필이 나열됩니다. 수집 프로필은 생성되는 라이브 렌 디션의 품질과 수를 제어합니다. 프로파일관리링크를 클릭하면 다른 브라우저 탭에서 인제스트 프로파일 편집기가 열립니다. 수집 프로필 편집기에 액세스하려면 계정 관리자 여야합니다. 클릭변환링크를 클릭하면 선택한 수집 프로필을 사용하여 생성될 변환을 볼 수 있습니다.
    세 계층의 비트 전송률 및 청구 세부 정보는 다음과 같습니다.
    • Tier 1은 최대 5,500Kbps를 허용하며 1.0x 시간에 청구됩니다.
    • Tier 2는 최대 10,000Kbps를 허용하며 1.5x 시간에 청구됩니다.
    • Tier 3는 최대 20,000Kbps를 허용하며 2.0x 시간에 청구됩니다.
  6. 완료되면 이벤트를 비디오 자산으로변환을 선택하여 라이브 비디오의 VOD 버전을 Video Cloud 계정에 저장합니다. 이 옵션을 선택하면 선택한라이브 수집 프로필의 최고 품질의변환이 VOD 비디오의 소스로 사용됩니다. 계정의 기본 수집 프로필은 변환을 만들 때 사용됩니다. 라이브 이벤트가 종료되는 즉시 VOD 버전을 사용할 수 없으며 처리 프로세스가 완료되면 동영상을 사용할 수 있습니다.
    참고: 반복 이벤트를 사용할 때는 이 옵션이 지원되지 않습니다.
  7. 고급 옵션을 클릭하고 고급 옵션을 구성합니다.
    참고: Beacon Live 작업의 경우 고급 옵션에서 스트림 상태 알림 확인란을 활성화해야 합니다.
  8. 스트림 상태 알림 활성화

    Beacon CMS의 알림 URL을 추가합니다 (이 URL은 Brightcove 지원에서 제공됨).

  9. 이벤트생성을 클릭하여 라이브 이벤트를 시작합니다. Control Room 페이지가 열리고 인코더에서 사용할스트리밍 엔드포인트 (RTMP URL)및스트림 이름을포함한 라이브 이벤트에 대한 세부 정보가 표시됩니다.

    스트림을 처리하는 데 약 1분 정도 소요됩니다.

    라이브 플랫폼에서 스트리밍을 시작하려면 인코더가 필요합니다. 인코더 사용 방법에 대한 정보는 다음 문서를 참조하십시오.

    단계별: Telestream Wirecast와 함께 라이브 모듈 사용

    단계별: 오픈 브로드 캐스터 소프트웨어 (OBS)와 함께 라이브 모듈 사용

  10. 에서제어실복사하다 "작업 아이디"그리고 "비디오 ID" ~로부터테스트 이벤트패널에서 Beacon 플랫폼 설정에서 라이브 이벤트를 구성하려면 둘 다 필요합니다.
  11. 그런 다음 Beacon으로 돌아가 이벤트 탭을 클릭하고 이전에 생성 한 이벤트로 이동 한 다음 기본 데이터 설정에서 라이브 이벤트의 작업 ID를 붙여 넣습니다.

    그런 다음 Streams탭으로 이동하여 새 스트림을 만들고스트림 비디오 ID에 이전에 만든비디오 클라우드 이벤트의 비디오 ID를붙여 넣습니다.

    다음 정보를 입력하십시오.

    • 비디오를 게시 할 수있는 장치 수를 선택하십시오.
    • 스트림 유형을다음과 같이 변경합니다. 라이브
    • 페이지를 떠나기 전에“이벤트 업데이트”버튼을 클릭하십시오.
  12. 캐시를 지 웁니다.

    Brightcove Beacon 앱으로 이동하여 재생 목록과 이벤트를 선택하십시오. 플레이를 클릭합니다 .

    이제 비콘 앱에서 비디오 클라우드의 라이브 스트리밍을 볼 수 있습니다.

    이벤트가 종료되면 클립으로 저장하여 나중에 Beacon에서 VOD로 전송할 수 있습니다.

Beacon Live 이벤트에서 클립 만들기

나중에 스트리밍할 비디오를 VOD 녹화 비디오로 유지하려면 Live Beacon 이벤트의 클립을 만들어야 합니다.

라이브 모듈은 완료된 이벤트에서 비디오 클립을 만드는 기능을 제공합니다. 라이브 이벤트가 진행되는 동안 클립을 만들 수도 있습니다.

전체 클리핑 기능 목록은 라이브 모듈을 사용하여 비디오 클립만들기설명서를 참조하십시오.

  1. 라이브 모듈에서 왼쪽 패널의 클립만들기탭으로 이동합니다.
  2. 클리핑 버튼 또는 단축키를 사용하여 클립의 시작 및 끝점을 설정합니다. 또는저장 버튼을클릭하기만 하면 전체 비디오를 유지할 수 있습니다.
  3. 클립 비디오 속성을 입력합니다.
    • 클립 이름 -클립 이름
    • 간단한 설명 -클립 설명
    • 태그 -클립과 연결할 태그
    • 참조 ID -클립에 할당 할 참조 ID
    • 소스 광고 시간 콘텐츠 제거 -이 옵션은 클립 내에서 트리거 된 광고 시간을 제거하고 프로그램 콘텐츠 만 유지합니다. 참고: 소스 광고 시간 제거 확인란은 스트림이 SSAI를 사용하는 경우에만 나타납니다.
  4. 클립만들기를 클릭합니다 . 클립에 대한 트랜스 코딩 프로세스가 시작됩니다. 클립은 계정의 기본 수집 프로필을 사용하여 트랜스 코딩됩니다.

클립을 재생 가능한 스트림 비디오로 Beacon에 추가하십시오.

  1. 번호를 받아 ID당신이 만든 클립의:
  2. Beacon 플랫폼에서이벤트탭을 선택하고스트림메뉴, 변경스트림 유형라이브에서 스트림으로의 드롭다운에서스트림 비디오 ID옵션, 붙여넣기 ID Live 플랫폼의 Clip 이벤트에서 방금 복사한 번호입니다.
  3. 기본 데이터탭을 클릭하고상태를종료됨에서완료로 변경합니다 . 페이지 하단의 이벤트업데이트버튼을 클릭하기 전에 앱에서재생버튼을 사용하여 이벤트 기록을 볼 수 있습니다.

(정적 진입 점) SEP 라이브 이벤트

SEP는 예를 들어 주간 프로그램과 같이 이벤트가 반복해서 시작될 때 활성화 / 비활성화 할 수있는 반복 이벤트입니다. 이 이벤트는 모두를위한 하나로 사용할 수 있습니다. 라이브 작업 ID Beacon 앱에서 여러 채널로 전송할 수 있습니다.

  • 되풀이 이벤트를 만들면 미디어 모듈에서 해당 이벤트에 대한 비디오가 자동으로 생성됩니다. 하다아니이벤트가 반복 될 때마다 필요하므로이 비디오를 삭제하십시오.
  • 인코더를 시작하기 전에 반복 이벤트를 활성화해야합니다.

반복 이벤트를 만들려면 라이브 이벤트를만들 때 정적 진입점 (SEP) 을 사용하여 반복 이벤트로 만들기옵션을 선택한 다음 이벤트만들기를 클릭합니다 .

  1. 라이브 플랫폼에서 새 이벤트를만듭니다 . 이벤트 생성 방법에 대한 자세한 내용은 다음문서를 참조하십시오 .
  2. 고급 옵션을확장하고“정적 진입점 (SEP) 을 사용하여 반복 이벤트로 만들기”옵션을 활성화합니다. SEP에 대한 자세한 내용을 보려면여기를 클릭하십시오.
  3. SEP 이벤트가 이미 있는 경우대기탭을 클릭하면 라이브 컨트롤 룸에서활성화/비활성화할 수 있습니다.
