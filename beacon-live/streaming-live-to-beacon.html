    비콘 앱으로 실시간 이벤트 스트리밍

    이 항목에서는 실시간 이벤트를 비콘 앱으로 스트리밍하는 방법을 배웁니다.

    개요

    라이브 이벤트를 애플리케이션으로 스트리밍하려면 일련의 단계에 따라 Video Cloud Live 계정과 브라이트코브 비콘을 연결해야 합니다.

    프로세스의 상위 수준 뷰는 다음과 같습니다.

    1. 첫 번째 단계는 브라이트코브 비콘에서 이벤트와 재생 목록을 만드는 것입니다. 이 이벤트는 앱에서 동영상의 자리 표시자로 사용할 수 있습니다.
    2. Video Cloud 계정으로 이동하여 Live에 액세스하여 이벤트를 만들고 식별자로 사용할 작업 ID를 얻습니다.
    3. 브라이트코브 비콘으로 돌아가면 라이브 스트리밍을 연결하여 라이브 플랫폼에서 생성된 작업 ID를 플랫폼의 이벤트 섹션 아래에 둘 수 있습니다.
    4. 연결을 기다리면 Beacon 앱에서 라이브 비디오가 스트리밍되기 시작합니다.
    5. 마지막 섹션에서는 정적 진입 점에 의해 라이브에서 다른 유형의 이벤트를 만드는 단계를 볼 수 있습니다.

    이 문서의 나머지 부분에서는 이러한 각 단계에 대한 세부 정보를 제공합니다.

    신호에서 라이브 이벤트 만들기:

    Brightcove Beacon 플랫폼에서이벤트탭을 클릭하여 이벤트 설정을 시작합니다.

    브라이트코브 비콘 라이브로활성화된 계정만 탭 섹션에서재생 목록및이벤트를사용할 수 있습니다. 활성화되지 않은 경우 브라이트코브 지원에 문의하십시오.
    1. 클릭새 이벤트 추가 :
    2. 라이브 이벤트에이름 , 그리고이벤트 시작일 :
    3. 에서텍스트 데이터탭에서 이벤트 정보를 입력하십시오.
    4. 에서비 텍스트 데이터탭에서 이벤트의 이미지 포스터를 선택하십시오.

      세로, 가로, 축소판, 와이드 배너 중에서 이벤트에 사용할 포스터 유형을 선택할 수 있습니다.

    5. 에서권리 및 일정탭에서 설정할 수 있습니다. 가용성 세부 정보같이:
      • 권리 유형 : 광고 , 구독 또는 무료 중에서 권리 유형을 선택합니다 .
      • 시작 시간: 이벤트 시작 시간
      • 종료 시간 : 이벤트의 종료 시간, 이 시간은 앱 내에서 제한을 생성하기 위해 만들어집니다.
      • 최대 스트림 : 앱에서 허용되는 최대 스트림 수입니다. 기본 숫자는 0 (영) 입니다.
      • 장치 세부 정보 : 여기에서 스트림을 게시할 다른 수의 장치를 추가할 수 있습니다. (예: iOS, 안드로이드, 로쿠, 삼성 등)
      • 지리적 위치 정보 : 앱을 사용하도록 설정할 국가 목록입니다. 국가를 제한하고 싶지 않을 경우 “World”를 선택할 수 있습니다.
      • 거부 된 위치 : 콘텐츠에 대한 액세스가 거부된 지역 목록입니다.
    6. 아래 버튼을 클릭하십시오: 새 이벤트 만들기

    재생 목록에 신호 이벤트 추가

    이벤트를 만들면재생 목록상단 메뉴의 탭. 우리는 우리가 뭔가를 스트리밍하기 전에 재생 목록에 이벤트를 추가해야합니다.

    1. 클릭새 재생 목록 추가버튼을 누르고 하나를 만듭니다.
      브라이트코브 비콘에서 라이브 이벤트를 포함할 재생 목록을 만들어야 합니다. 하지 마라 Video Cloud의 재생 목록을 사용하면 Video Cloud 재생 목록을 사용하면 Beacon Live 녹화를 덮어 쓰거나 삭제할 수 있습니다.
    2. 에재생 목록 , 콘텐츠 유형 탭에서이벤트탭:
    3. 콘텐츠 모듈에서 이벤트를 찾은 다음 (+) 버튼을 클릭하여 왼쪽 재생 목록에 추가합니다. 이벤트 순서를 앱에 표시하려는 방식으로 배치할 수 있습니다.
    4. 재생 목록 (시간, 장치, 위치 등) 의 설정 조정 당신은 당신의 이벤트와 함께했던 것처럼.
    5. 재생 목록 버튼에 변경 사항 저장버튼을 클릭합니다.
    6. 화면 오른쪽 상단에있는 도구 버튼을 클릭하여 캐시를 지우면 모든 것이 앱에 올바르게 표시됩니다.

    이 시점에서 이벤트는 브라이트코브 비콘 앱에 컨테이너로 생성되지만 실제로 라이브 비디오 스트리밍을 시작하려면 Video Cloud Live Platform에서 전송을 설정해야 합니다.

    Studio에서 라이브 이벤트 만들기

    라이브 모듈에 액세스하려면 Video Cloud Studio에 로그인하고 탐색 헤더에서 Live를 클릭합니다.

    1. 이벤트 관리 페이지에서이벤트 생성 . 라이브 이벤트 만들기 또는 라이브 채널 만들기 페이지가 열립니다. 이벤트와 채널을 만들 때의 속성은 동일합니다. 이 문서의 스크린샷은 이벤트 생성에 대해 설명합니다.
    2. 이벤트 상세내역을 입력합니다.
      • 이벤트 이름 - 이벤트 이름입니다. 이 이름은 비디오가 비디오 자산으로 저장된 경우 미디어 모듈에도 나타납니다 (최대 255자).
      • 간단한 설명 - 이벤트 설명 (최대 250자)
      • 태그 - 이벤트와 연결할 태그입니다. 비디오가 비디오 자산으로 저장된 경우 태그가 비디오에 할당됩니다 (최대 128자 및 1200 태그).
      • 참조 ID - 이벤트의 참조 ID입니다. 고유해야 합니다 (최대 150자).
    3. 계정에 대한 모든 필수 사용자 정의 필드가 표시됩니다. 사용자 정의 필드에 값을 할당합니다.
    4. 소셜 미디어로 스트리밍을허용됨으로설정하면 이벤트를 소셜 미디어 플랫폼으로 스트리밍할 수 있습니다. 이것은 할 수 있습니다이벤트 스트리밍이 시작되면또는 사용하여소셜 모듈 .
    5. 라이브 인제스트 프로파일을선택합니다. 드롭다운 목록에 모든 라이브 인제스트 프로필이 나열됩니다. 인제스트 프로파일은 작성되는 라이브 변환의 품질과 수를 제어합니다. 프로파일 관리링크를 클릭하면 다른 브라우저 탭에서 인제스트 프로파일 편집기가 열립니다. 인제스트 프로파일 편집기에 액세스하려면 계정 관리자여야 합니다. 선택한 인제스트 프로파일을 사용하여 작성될 변환을 보려면 RENDITION스 링크를 클릭합니다.
      세 계층의 비트율 및 청구 세부 정보는 다음과 같습니다.
      • 계층 1은 최대 5,500Kbps를 허용하며 1.0x 시간으로 청구됩니다.
      • 계층 2는 최대 10,000Kbps를 허용하며 1.5x 시간으로 청구됩니다.
      • 계층 3은 최대 20,000Kbps를 허용하며 2.0x 시간으로 청구됩니다.
    6. 완료되면 이벤트를 비디오 자산으로 변환을선택하여 라이브 비디오의 VOD 버전을 Video Cloud 계정에 저장합니다. 이 옵션을 선택하면 선택한라이브 인제스트 프로파일의최고 품질의 변환이 VOD 비디오의 소스로 사용됩니다. 변환을 만들 때 계정의 기본 인제스트 프로필이 사용됩니다. 라이브 이벤트가 종료되는 즉시 VOD 버전을 사용할 수 없습니다. 수집 프로세스가 완료되면 비디오를 사용할 수 있습니다.
      참고: 반복 이벤트를 사용할 때는 이 옵션이 지원되지 않습니다.
    7. 고급 옵션을 클릭하고 고급 옵션을 구성합니다 .
      참고: 실시간 비콘 작업의 경우 고급 옵션에서 스트림 상태 알림 확인란을 활성화해야 합니다.
    8. 스트림 상태 알림 활성화

      비콘 CMS에서 알림 URL을 추가합니다 (이 URL은 브라이트코브 지원에서 제공됨).

    9. 이벤트 만들기를클릭하여 라이브 이벤트를 시작합니다. 인코더에서 사용할스트리밍 끝점 (RTMP URL)및스트림 이름등 라이브 이벤트에 대한 세부 정보가 포함된제어실 페이지가열립니다.

      스트림을 처리하는 데 약 1 분이 소요됩니다.

      라이브 플랫폼에서 스트리밍을 시작하려면 인코더가 필요합니다. 인코더 사용 방법에 대한 자세한 내용은 다음 설명서를 참조하십시오.

      단계별: 텔레스트림 와이어캐스트와 함께 라이브 모듈 사용

      단계별: 오픈 브로드캐스터 소프트웨어 (OBS) 와 함께 라이브 모듈 사용

    10. 에서제어실복사 "작업 ID"그리고“동영상 ID” ~로부터테스트 이벤트패널에서 Beacon 플랫폼 설정에서 라이브 이벤트를 구성하려면 둘 다 필요합니다.
    11. 그런 다음 비콘으로 돌아가서 이벤트 탭을 클릭하고 이전에 만든 이벤트로 이동한 다음 기본 데이터 설정에서 라이브 이벤트의 작업 ID를 붙여 넣습니다.

      그런 다음스트림탭에서 새 스트림을 만듭니다. 스트림 비디오 ID , 붙여 넣기비디오 ID이전에 만든 Video Cloud 이벤트에서

      다음 정보를 입력합니다.

      • 비디오를 게시할 수 있는 장치 수를 선택합니다.
      • 변경스트림 유형에: 라이브
      • 그리고“이벤트 업데이트”페이지를 떠나기 전 버튼 :
    12. 캐시를 지웁니다.

      브라이트코브 비콘 앱으로 이동하여 재생 목록과 이벤트를 선택합니다. 클릭플레이 .

      이제 비콘 앱에서 비디오 클라우드에서 라이브 스트리밍을 볼 수 있습니다.

      이벤트가 종료되면 클립으로 저장하여 나중에 신호의 VOD 로 전송할 수 있습니다.

    비콘 라이브 이벤트에서 클립 만들기

    나중에 스트리밍하기 위해 비디오를 VOD 녹화 비디오로 유지하려면 라이브 비콘 이벤트의 클립을 만들어야 합니다.

    라이브 모듈은 완료된 이벤트에서 비디오 클립을 만드는 기능을 제공합니다. 라이브 이벤트가 진행되는 동안에도 클립을 만들 수 있습니다.

    전체 클리핑 기능 목록을 보려면라이브 모듈을 사용하여 비디오 클립 만들기설명서를 참조하십시오.

    1. 라이브 모듈에서클립 생성왼쪽 패널의 탭
    2. 클리핑 버튼 또는 바로 가기 키를 사용하여 클립의 시작점과 끝점을 설정합니다. 또는저장 버튼전체 비디오를 유지합니다.
    3. 클립 비디오 속성을 입력합니다.
      • 클립 이름 - 클립의 이름
      • 간단한 설명 - 클립에 대한 설명
      • 태그 - 클립과 연결할 태그
      • 참조 ID - 클립에 할당할 참조 ID
      • 소스 광고 중단 콘텐츠 제거 - 이 옵션은 클립 내에서 트리거된 광고 나누기를 제거하여 프로그램 콘텐츠만 유지합니다. 참고: 소스 광고 중단 제거 확인란은 스트림이 SSAI를 사용하도록 설정된 경우에만 나타납니다.
    4. 클립 만들기를클릭합니다. 클립의 코드 변환 프로세스가 시작됩니다. 클립은 계정의 기본 인제스트 프로필을 사용하여 코드 변환됩니다.

    클립을 비콘에 재생 가능한 스트림 비디오로 추가합니다.

    1. 번호 가져가신분증생성 된 클립의 :
    2. Beacon 플랫폼에서이벤트탭을 선택하고스트림메뉴에서스트림 유형라이브에서 스트림으로의 드롭 다운 및스트림 비디오 ID옵션, 붙여 넣기신분증 Live 플랫폼의 Clip 이벤트에서 방금 복사 한 번호입니다.
    3. 기본 데이터탭을 클릭하고상태를종료에서완료로변경합니다. 떠나기 전에이벤트 업데이트페이지 하단의 버튼을 클릭하면플레이이벤트 녹화를 보려면 앱에서 버튼을 사용할 수 있습니다.

    (정적 진입점) SEP 라이브 이벤트

    SEP는 주간 프로그램과 같이 이벤트가 반복해서 시작될 때 활성화/비활성화할 수 있는 반복 이벤트입니다. 이 이벤트는 모두를위한 하나로 사용할 수 있습니다. 라이브 작업 ID Beacon 앱에서 여러 채널로 전송할 수 있습니다.

    • 반복 이벤트를 만들면 Media 모듈에서 해당 이벤트에 대한 비디오가 자동으로 만들어집니다. 이 비디오는 이벤트가 재발될 때마다 필요하므로 삭제하지 마십시오.
    • 인코더가 시작되기 전에 반복 이벤트를 활성화해야 합니다.

    반복 이벤트를 만들려면 라이브 이벤트를만들 때 [정적 진입점 (SEP) 을 사용하여 반복 이벤트로만들기] 옵션을 선택한 다음 [ 이벤트 만들기 ] 를 클릭합니다.

    1. 라이브 플랫폼에서새로운 이벤트 생성 . 이벤트 생성 방법에 대한 자세한 내용은 다음을 참조하십시오. 선적 서류 비치 .
    2. 확장고급 옵션 , 활성화 "정적 진입 점 (SEP)을 사용하여이 이벤트를 반복 이벤트로 생성"선택권. SEP에 대한 자세한 내용을 보려면여기.
    3. 이미 SEP 이벤트가있는 경우대기탭, 당신은 할 수 있습니다활성화 / 비활성화라이브 관제실에서 가져옵니다.
    4. 복사라이브 작업 ID
    5. 브라이트코브 비콘 플랫폼으로 돌아가서 이름과 예약된 시간을 제공하는 이벤트를 구성합니다.
    6. 이벤트 날짜 및 시간은 다음 형식이어야 합니다.
    7. 이벤트 업데이트버튼을 클릭합니다.

    브라이트코브 비콘 플랫폼에서 원하는 모든 이벤트 채널을 사용하여 이 프로세스를 수행할 수 있으며, 모든 이벤트 채널은 동일한 라이브 이벤트 전송에서 스트리밍되지만 각 이벤트 설정에서 조정한 날짜 및 시간이 서로 다릅니다.
