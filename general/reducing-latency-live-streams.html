라이브 스트림의 지연 시간 줄이기

이 항목에서는 Brightcove Live 스트림의 지연 시간을 줄이는 방법을 설명합니다.

소개

지연 시간은 무언가가 실제 생활에서 녹화 된 시점과 비디오 플레이어에서 보이는 시점 사이의 시간 지연입니다. 물론 소스에서 인터넷을 통해 시청자 플레이어에게 신호를 전송하는 데 필요한 실시간 신호가 있습니다. 지연 시간은 온라인 비디오가 세그먼트라고하는 신중한 청크로 전송된다는 사실로 인해 악화됩니다.

비디오 세그먼트의 길이를 줄여 지연 시간을 줄일 수 있으며 다음 섹션에서는이를 수행하는 방법을 자세히 설명합니다. 그러나 중단없는 재생을 보장하기 위해 플레이어는 현재 재생 위치 (버퍼링이라고 함)보다 앞서 몇 개의 비디오 세그먼트를 다운로드하여 필요할 때 준비 할 수 있습니다. 세그먼트 길이를 줄이면 플레이어가 더 적은 비디오 데이터를 미리 가져 오며 플레이어가 추가 세그먼트를 버퍼링하는 동안 재생이 일시 중지 될 가능성이 높아집니다.

추가 제한

지연 시간을 줄이는 라이브 스트림에는 다음 제한이 적용됩니다.

  • SSAI는 ~ 아니다지원
  • DRM은 ~ 아니다지원
  • DVR 창은 5분 미만이어야 합니다.
  • 이상 4번역

지연 시간 단축 라이브 프로필 생성

지연 시간을 줄이려면 사용자 지정 Live 프로필을 만들어야합니다. 아래 단계를 따르십시오.

  1. Studio에 로그인
  2. 관리 > 인제스트 프로필로 이동합니다 .
  3. 다음 중 하나를 선택하십시오. 살다이하의 프로필 수집 4변환:
  4. 링크를 클릭하여 프로필을 엽니 다.
  5. Duplicate버튼을 클릭하여 이 프로파일을 기반으로 새 프로필을 만듭니다.
  6. 새 프로필에서name를 “낮은 지연 시간 프로파일" (또는 호출하려는 모든 프로파일) 으로 변경합니다.
  7. 각 에renditions대해 를segment_seconds2로 변경하여세그먼트 길이를 2초까지
  8. 전체 프로필은 다음과 같아야합니다.

Live API 사용

대기 시간을 줄이고 라이브 API를 사용하여 라이브 작업을 생성하는 경우 평소와 같이 작업을 생성하되 다음을 포함합니다. 4회 이하 , 설정segment_seconds각 변환에 대해2 . 또한소개에 나열된 다른 제한 사항을 준수해야합니다.
