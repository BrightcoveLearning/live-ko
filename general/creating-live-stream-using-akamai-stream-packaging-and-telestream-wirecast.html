페이지 내용

    Akamai 스트림 패키징 및 텔레스트림 와이어캐스트를 사용하여 라이브 스트림 생성

    이 주제에서는 Akamai Stream Packaging 및 Telestream Wirecast를 사용하여 라이브 스트리밍 이벤트를 브로드 캐스트하는 방법에 대해 알아 봅니다.

    이 주제는 Akamai Stream Packaging을 사용하여 라이브 이벤트를 방송하는 데 도움이됩니다.   Telestream Wirecast 인코더, 원격 자산 비디오 및 Brightcove Player. 다른 인코더 역시 사용할 수 있습니다. 브라이트코브에서 라이브 스트리밍 비디오 제공에 사용할 수 있는 모든 옵션 목록을 보려면 라이브 스트림 제공을 참조하십시오.

    Akamai Stream Packaging을 사용하여 라이브 스트림을 방송하려면 다음을 수행하십시오.

    • 라이브 스트리밍을 위해 Video Cloud 계정 활성화
    • 미디어 모듈을 사용하여 라이브 스트림 비디오 만들기
    • 라이브 이벤트를위한 Telestream Wirecast 구성
    • 라이브 이벤트에 대한 게시 코드 생성

    라이브 스트리밍을 위해 비디오 클라우드 계정 활성화

    라이브 스트리밍 이벤트를 실시하기 전에 브라이트코브 지원에 연락하여 라이브 스트리밍을 위해 사용자의 비디오 클라우드 계정 활성화를 원한다고 알려야 합니다. 또한 해당 계정에 대해 스트림 패키지 구성 활성화를 요청해야 합니다. 계정이 구성되면 라이브 스트리밍을 설정하는 데 필요한 자격 증명 및 구성 정보가 포함 된 이메일을 받게됩니다.

    참고: Akamai HD용 자격 증명은 사용자의 인코더 IP에 따라 지리적으로 제공됩니다. 새 위치에서 이벤트를 계획할 경우 새 위치의 인코더에 대해 Ping할 수 있는 외부 IP 주소를 브라이트코브 지원에 제공해야 합니다. 그래야 Akamai가 진입점을 가장 가까운 위치에서 다시 제공하여 최대한 공용 인터넷을 통한 이동을 줄이고, 지연 시간과 중단 위험을 낮출 수 있습니다. 변경 사항이 Akamai 네트워크에 전달되기까지 얼마간의 시간이 소요되기 때문에 브라이트코브 지원은 사전에 이 사실을 파악해야 합니다. 인코더가 아직 위치에 없어도 사용할 네트워크에 Ping할 수 있는 외부 IP 주소만 있으면 문제될 것이 없습니다.

    미디어 모듈을 사용하여 라이브 스트림 비디오 생성

    라이브 스트림을 취급하려면 사용자의 비디오 라이브러리에 있는 원격 자산을 사용하여 비디오를 생성해야 합니다. 원격 자산은 비디오 파일을 사용자의 자체 시스템또는 제3자 시스템에서 유지 관리할 때 사용됩니다. 그런 다음 비디오를 생성할 때 브라이트코브에 비디오 파일의 URL을 제공합니다. 원격 자산은 미디어 모듈이나 CMS API를 사용하여 생성됩니다.

    미디어 모듈을 사용하여 원격 자산 비디오를 생성하려면 다음 단계를 따르십시오.

    1. 미디어 모듈을 엽니다.
    2. 딸깍 하는 소리 + 원격 비디오 추가측면 탐색 패널에서.
    3. 비디오 이름을 입력합니다.
    4. 비디오 유형을 라이브로 설정합니다.
    5. 설정체재 ...에 HLS .
    6. 에 대한변환 URL , 시작 ARL Brightcove 지원에서 귀하의 이메일에 제공된 값. ARL URL은 다음 형식입니다. 
       http://bcliveunivsecure-lh.akamaihd.net/[EVENT_ANGLE_BITRATE]@684348
      당신은이벤트와 각도값이있는 자리 표시 자. BITRATE사용되지 않습니다. 예를 들어 URL은 다음과 같습니다. 
       http://bcliveunivsecure-lh.akamaihd.net/Live_1@684348

      삽입나는다음 URL에서 akamaihd.net URL의 일부 :

       http://bcliveunivsecure-lh.akamaihd.net/i/Live_1@684348

      또한 추가 /master.m3u8 URL의 끝에. 최종 비디오 버전 URL은 다음과 유사해야 합니다.

      http://bcliveunivsecure-lh.akamaihd.net/i/Live_1@684348/master.m3u8
    7. 형식이 지정된 URL을 URL필드를 클릭하고변환 추가 .
    8. 저장을 클릭하여 원격 자산 비디오를 저장합니다.
    9. 원격 자산이 미디어 모듈에 표시되는지 확인합니다. 필요한 경우 원격 자산 비디오를 활성화합니다.

    미디어 모듈에서 원격 자산 비디오를 생성하면 기타 모든 비디오와 마찬가지로 해당 비디오의 메타데이터를 편집할 수 있습니다.

    라이브 이벤트를 위해 Telestream Wirecast 구성

    이 섹션에서는 라이브 스트리밍 이벤트를 지원하기 위해 Telestream Wirecast 인코더를 구성 및 사용합니다. Telestream Wirecast 소프트웨어는 카메라로부터 입력을 캡처하여 CDN에서 전달할 수 있는 스트림을 생성하는 데스크톱 애플리케이션입니다. 라이브 스트리밍 이벤트를 전달하는 데 더욱 적합할 수 있는 하드웨어 및 기타 소프트웨어 기반 인코딩 솔루션이 있을 수 있습니다. Telestream 웹사이트에서 Wirecast 소프트웨어 무료 평가판 버전을 다운로드할 수 있습니다.

    참고: 계속하기 전에 컴퓨터에 Wirecast 소프트웨어가 설치되어 있고 카메라가 연결되어 있어야 합니다.

    라이브 이벤트를 위해 Wirecast를 구성하려면 다음 단계를 따르십시오.

    1. Wirecast를 엽니다.
    2. +를 클릭하고비디오 캡처를선택합니다.
    3. 목록에서 카메라를 선택합니다. 추가를 클릭합니다.
    4. 선택한 비디오 소스가 미리 보기 창에 나타나는지 확인합니다.
    5. 화살표 버튼 ( ) 카메라가 라이브 샷으로 촬영되도록합니다.
    6. 스트림 버튼 ( ). 출력 설정을 입력하라는 메시지가 나타납니다.
    7. 대상을 RTMP 서버로 설정합니다. 확인을 클릭합니다.
    8. 에 대한주소 , 사용기본 진입 점 Brightcove 지원 이메일에 제공된 값.
    9. 에 대한흐름 , 원격 자산 URL을 정의 할 때 사용 된 [EVENT_ANGLE_BITRATE] @ 394443 값을 사용하십시오. 원격 자산을 만들 때 사용한 EVENT 및 ANGLE에 동일한 값을 사용해야합니다. BITRATE 값을 Wirecast에 설정된 비트 전송률로 설정합니다.
    10. 자격 증명 설정...을 클릭합니다.
    11. 브라이트코브 지원에서 보낸 이메일에 제공된 사용자 이름 및 암호를 입력합니다. 확인을 클릭합니다.
    12. 확인을 클릭하여 출력 설정을 저장합니다.
    13. 스트림 버튼을 클릭합니다. 라이브 스트림이 시작되어야 합니다.

    라이브 이벤트를 위한 게시 코드 생성

    1. 미디어 모듈을 엽니다.
    2. 라이브 스트림 원격 자산 비디오를 선택합니다.
    3. 게시 및 포함을클릭합니다.
    4. 플레이어를 선택합니다.
    5. 플레이어 URL을 클릭하여 브라우저에서 라이브 스트림을 미리 봅니다.
