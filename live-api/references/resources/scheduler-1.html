잡스타트스톱

JobStartStop 엔드 포인트를 사용하여 기존 라이브 작업의 활성화 및 비활성화를 예약 할 수 있습니다. 지정된 시간에 Google 시스템은 사용자를 대신하여 작업을 활성화 및 비활성화하는 데 필요한 API 호출을 수행합니다.

요구 사항

  • 계정의 라이브 API 키입니다.
  • 라이브 계정에는 스케줄러 재정의가 활성화되어 있어야합니다.
  • “정적”이 있는 기존 라이브 잡: true입니다.

작업스타트스톱 워크플로우 생성

2021-02-02T14:00:00 + 00:00 에서 시작되는 2시간 이벤트를 예약하고 싶다고 가정해 보겠습니다 .

타임 스탬프는 초 단위의 unix 타임 스탬프로 UTC 형식이어야합니다. 당신이 사용할 수있는 https://www.unixtimestamp.com/변환을 위해.

여기에 설명된 활성화 시간 제한은 예약된활성화에도 여전히 적용되므로 원하는 시작 시간 60초 전에 시간을 예약하는 것이 좋습니다.

이 예에서는 예정된 시작 시간 5 분 전을 선택합니다.

시작: 2021-02-02T13:55시+00시 = 1612274100 정거장: 2021-02-02T16:00시+00시 = 1612281600년

워크 플로우를 작성하려면 다음 요청을 작성하십시오.

POST https://api.bcovlive.io/v1/scheduler/jobstartstop 헤더: 콘텐츠 유형: 응용 프로그램/json X-API-키: API 키 본문: { "job_id": "your Job ID", "activation": { "time_utc": 1612274100 }“비활성화”: { "time_utc": 1612281600 } , “설명”: “내 화요일 방송”}

다음과 유사한 응답을 받아야합니다.

{ "account_id": "your Account ID", "activation": { "state": "pending", "time_utc": 1612274100 } , “비활성화”: { "state": "pending", "time_utc": 1612281600 } , “설명”: “내 화요일 방송”, “작업 ID”: “작업 ID”, “유형”: “작업 시작 중지”, “워크플로_ID”: “d44cf29f4f184757ac0995beb8a0097e”}

작업스타트스톱 워크플로 업데이트

JobStartStop 워크 플로우의 활성화 및 비활성화 시간을 업데이트 할 수 있습니다. “상태”: “보류 중”인 경우에만 업데이트할 수있습니다.

이전 예제에 따라 예정보다 30 분 일찍 이벤트를 중지하려고합니다.

PUT https://api.bcovlive.io/v1/scheduler/jobstartstop/workflow_id 헤더: 콘텐츠 유형: 응용 프로그램/json X-API-키: API 키 본문: { "deactivation": { "time_utc": 1612279800 } }
