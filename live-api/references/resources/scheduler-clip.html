클립

클립 끝점을 사용하면 기존 라이브 작업의 클립 이벤트를 예약 할 수 있습니다. 지정된 시간에 당사 시스템은 사용자를 대신하여 클립 작업을 시작하는 데 필요한 API 호출을 수행합니다.

요구 사항

  • 계정의 라이브 API 키입니다.
  • 라이브 계정에는 스케줄러 재정의가 활성화되어 있어야합니다.

클립 작업 과정 만들기

클립을 생성할 때 클립 시간 경계 ( 출력필드) 와 클립이 요청되도록 스케줄링된 시간 ( time_utc필드) 을 모두 지정해야 합니다.

요청 본문의출력필드는 표준 VOD 끝점에서출력필드의 동일한 필드를 모두 지원하지만 예약된 클립은 목록이 아닌 단일 출력만 지원합니다. 보다 VOD 클리핑 지원 문서사용 가능한 필드에 대한 자세한 내용은

일반적으로 다음을 설정합니다. time_utc클립의 끝 경계와 일치하도록 합니다. 예약 된 시간은 클립의 끝 경계 이후 여야합니다. 그렇지 않으면 예약 된 시간에 클립 요청이 실패합니다.

time_utc필드의 타임스탬프는 유닉스 타임스탬프로 형식화된 UTC여야 합니다. 당신이 사용할 수있는 https://www.unixtimestamp.com/변환을 위해.

2021-02-09T13:55:00 + 00:00 에서 시작할 15분 클립을 예약하고 싶다고 가정해 보겠습니다 . output.start_time과 output.end_time을 사용하여 클립 경계를 유닉스 시간 (초) 으로 지정할 수 있습니다. (다시, 참조 VOD 클리핑 지원 문서사용 가능한 필드에 대한 자세한 내용은).

시작 시간: 2021-02-09T13:55:00 + 00:00 = 1612878900 종료 시간: 2021-02-09T14:10시+00시 = 1612879800

스케줄러는 클립 요청을 예약하는 데 시간이 필요하기 때문에 end_time과 동일한 값을 초 단위로 유닉스시간이기 때문에 사용할 수 있습니다.

또한 클립에 대한 Videocloud Video를 만들어출력에 포함시키려고합니다. 워크 플로를 예약 할 때 Videocloud 비디오가 생성되고 클립이 만들어지기 전에 비디오 ID를 알 수 있도록 해당 ID가 응답에 반환됩니다. 나중에 클립 요청을 취소하거나 예정된 시간에 실패하면 Videocloud Video가 계정에서 제거되지 않습니다.

워크 플로우를 작성하려면 다음 요청을 작성하십시오.

POST https://api.bcovlive.io/v1/scheduler/clip 헤더: 콘텐츠 유형: application/json X-API-KEY: API 키 본문: { "job_id": "your Job ID", "description": "My Scheduled Clip", "clip": { "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "name": "My Scheduled Clip" } } }, "time_utc": 1612879800}}

다음과 유사한 응답을 받아야합니다.

{ "account_id": "your Account ID", "description": "My Tuesday Clip", "type": "clip", "job_id": "your Job ID", "clip": { "state": "pending", "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" } }}, “타임_UTC”: 1612879800} “워크플로_ID”: “D44CF29f4F184757AC0995베바8A0097e”,}

클립.output.videocloud.video.id에는 새로 만든 비디오클라우드 비디오의 ID가 포함되어 있습니다. time_utc에서는 비디오 소스가 인제스트된 클립으로 업데이트됩니다.

클립 작업 과정 업데이트

클립의출력과 time_utc를 모두 업데이트할 수 있습니다. “상태”: “보류 중”인 경우에만 업데이트할 수있습니다.

이전 예에 따라 클립이 계획된 것보다 5 분 짧아 지길 원합니다. 출력필드를 업데이트할 때는 변경 사항뿐만 아니라 전체 객체를 제공해야 합니다. 여기에는 위의 create 호출에서 반환 된 Videocloud 비디오 ID가 포함됩니다.

이 예에서는 두 가지 모두 업데이트했습니다. end_time그리고 time_utc 5분 단축을 반영합니다.

PUT https://api.bcovlive.io/v1/scheduler/clip/workflow_id 헤더: 콘텐츠 유형: application/json X-API-KEY: API 키 본문: { "clip": { "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879500, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" } } }, "time_utc": 1612879500}}

응답으로 업데이트 된 워크 플로를 받게됩니다.

{ "account_id": "your Account ID", "description": "My Tuesday Clip", "type": "clip", "job_id": "your Job ID", "clip": { "state": "pending", "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879500, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" } }}, “타임_UTC”: 1612879500} “워크플로_ID”: “D44CF29f4F184757AC0995베바8A0097e”,}

클립 작업 과정 취소

DELETE요청으로 클립을 취소할 수 있습니다.

Clip 워크 플로를 취소하면 예약시 생성 된 모든 Videocloud 비디오가아니백엔드에 의해 제거됩니다. Videocloud 계정에서 분리 된 비디오를 수동으로 제거해야합니다.

삭제 https://api.bcovlive.io/v1/scheduler/clip/workflow_id 헤더: X-API 키: API 키

응답으로 업데이트 된 워크 플로를 받게됩니다.

{ "account_id": "your Account ID", "description": "My Tuesday Clip", "type": "clip", "job_id": "your Job ID", "clip": { "state": "cancelled", "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879500, "videocloud": { "video": { "id": "70702010152202", "name": "My Scheduled Clip" } }}, “타임_UTC”: 1612879500} “워크플로_ID”: “D44CF29f4F184757AC0995베바8A0097e”,}

알림

워크 플로를 만들 때 선택적으로 알림을 구성 할 수 있습니다. POST를 수행할 서비스에 대한 URL을제공해야 합니다. 스케줄러가 클립 호출을 할 때 알림을 보냅니다. 클립 N초 전에 알림을 받도록 구성할 수도 있습니다. 자세한 내용은알림 안내서를참조하십시오.
