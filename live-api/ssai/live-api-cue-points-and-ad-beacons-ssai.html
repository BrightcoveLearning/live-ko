개요

서버 측 광고 삽입 (SSAI) 을사용하면 지정된 시간에 라이브 스트리밍 이벤트 중에 광고를 표시할 수 있습니다. 일반적인 정보는 Live API를 참조하십시오. 서버 측 광고 삽입 (SSAI) 문서

  • SSAI를 사용하려면 먼저 계정에 대해 SSAI를 사용하도록 설정해야 합니다. 자세한 내용은 계정 관리자에게 문의하십시오.

큐 포인트

광고 시간은 큐 포인트에 의해 실행되며 두 가지 방법으로 지정할 수 있습니다.

  • 인코더에서 Brightcove로 보냄
  • 라이브 API를 통해 생성된 즉각적인 큐 포인트

인코더에서

Brightcove 실시간 전달 시스템은 인코더가 제출 한 큐 포인트를 AMF 형식으로 해석 할 수 있습니다.

  AMFDataList
  [0]:onCuePoint
  [1]:{Obj[]:
    time: 1.9889, //Difference from PTS of THIS packet to the first PTS of the 1st video frame in the adbreak
    name: "scte35",
    type: "event",
    ad_server_data: "YWJjZGVmZ2g=",	// optional introduced by Brightcove. It is a base64 encoded json map of parameters e.g. {‘key’:’value’}
    parameters: {Obj[]:
      type: "avail_in",
      duration: 12.0
    }
  }

메모:

  • 현재 RTMP 입력에서는avail_in타입 큐 포인트만 지원됩니다.
  • SCTE-35 큐 포인트는 RTP 및 SRT 입력에서 지원됩니다.
참조아래 부록 Elemental 인코더에 대한 샘플 큐 포인트 설정의 스크린샷.

수동 큐 포인트 삽입

다음을 사용하여 즉각적인 큐 포인트를 만들 수 있습니다. 라이브 API를 보내서POST요구:

방법 POST
URL https://api.bcovlive.io/v1/jobs/Job_ID/cuepoint
머리글 X-API-KEY: your API KEY

다음을 지정하는 요청 본문을 포함합니다.

들 유형 기술
duration 정수 휴식 시간 (초)입니다.

삽입되는 큐 포인트의 지속 시간은세그먼트 길이의 최소 2배일에. 기간 예제를 참조하십시오 .
timecode SMPTE 형식 선택 과목: SMPTE 형식의 시간 코드 HH:MM:SS:FF (FF = 프레임) 로, 변수 집합 (키/값 쌍) 을 ADServer로 전달해야 하는 시기를 지정합니다.

생략하면 큐 포인트가 즉시 삽입됩니다.

timecode 속성을 사용하는 경우 인코더는 SMPTE 형식( HH:MM:SS:FF )에 저장된 타임코드tc재산을 통해OnFI . 타임 코드는 라이브 스트림의 시작부터입니다.
ad_server_data 목적 선택 과목: 전달하는 키 / 값 쌍은 사용중인 광고 서버에 따라 다릅니다. 자세한 내용은 광고 서버 설명서와 광고 매크로를 사용한타겟팅 광고섹션을 참조하십시오.

기간 예

삽입되는 큐 포인트의 지속 시간은세그먼트 길이의 최소 2배일에.

예를 들어 를 사용하여 작업에 10초 큐 포인트를"segment_seconds"=4삽입하면 정상적으로 작동합니다. 그러나 와 같은 큐 포인트를 작업에"segment_seconds"=6삽입하면 다음 오류가 발생합니다.

  "error": "The parameter duration should be greater than
    or equal to (2 * target duration) of the job"
 

샘플 요청 본문

  {
    "duration": 30,
    "timecode": "15:50:49:16",
    "ad_server_data" : {
    "adbreakid": 12312
    "breaktheme": "fitness"
    }
  }

메모

  1. Wirecast 및 OBS와 같은 소프트웨어 인코더는 다음을 통한 전송 타임 코드를 지원하지 않습니다. OnFI RTMP 스트림의 패킷
  2. Elemental 하드웨어 인코더는 다음을 통해 전송 타임 코드를 지원합니다. OnFI RTMP 스트림의 패킷

비콘

비콘은 광고 재생 여부와 재생 횟수를 추적하기 위해 타사 분석으로 전송되는 재생 데이터 포인트입니다. 이 섹션에서는 Live API를사용하여 설정할 수 있는 비콘 유형과 데이터를 제공하는 데 사용할 수 있는 변수를 살펴보겠습니다. 다음 섹션에서는 비콘 세트를 만들고 관리하는 데 사용하는 API 요청에 대해 자세히 설명합니다.

비콘 유형

비콘 유형
비콘 유형 기술
Load 세션 당 한 번 실행되고 최상위 매니페스트가 요청 될 때만 트리거됩니다.
Play 콘텐츠가 요청되었으며 첫 번째 세그먼트가 반환되었습니다.

SCTE-35 광고 변수

SCTE (케이블 통신 엔지니어협회) 는 라이브 스트림에 대한 동적 광고 삽입에 대한 표준을정의합니다. 다음 표에는 SCTE-35 광고 구성 변수가 요약되어 있습니다.

SCTE-35 광고 매개 변수는 현재 FEC (Forward Error Correction)가 있거나없는 RTP (Real-time Transport Protocol) 및 TS (Transport Stream)를 기반으로하는 SRT (Secure Reliable Transport) 입력 소스 모두에서 지원됩니다. RTP에는 FEC를 적극 권장합니다.

이러한 광고 매개 변수는 RTMP (실시간 메시징 프로토콜) 를 사용하는 입력 스트림에서는 사용할 수 없습니다.
SCTE-35 광고 구성 변수
변하기 쉬운
기술
scte35_eventID
SCTE35 메시지와 함께 전달된 고유 이벤트 ID입니다.
scte35_programID
SCTE35 메시지와 함께 전달 된 고유 프로그램 ID입니다.
scte35_availNum
광고에 사용할 수있는 특정 접속 시간에 대한 ID이며 SCTE35 메시지를 통해 전송됩니다.
scte35_breakDuration
SCTE35 메시지와 함께 전달 된 프로그램의 90kHz 클록 틱으로 표시되는 광고 시간의 휴식 시간입니다.
scte35_spliceTime
SCTE35 메시지와 함께 전달 된 프로그램의 90kHz 클럭의 틱으로 표시되는 광고 시간의 연결 시간입니다.

ID로 설정된 단일 신호를 검색하려면GET요청을 제출하십시오.

방법 GET
URL https://api.bcovlive.io/v1/ssai/beaconsets/beaconset/beacon_set_id
머리글 X-API-KEY: your API KEY

샘플 응답

  {
      "account_id": "User account ID",
      "beacon_set_id": "Beacon set ID",
      "beacon_urls": [{
        "beacon_type": "Load",
        "beacon_url": "https://myserver.com/beaconRX2/load"
    },
    {
      "beacon_type": "Play",
      "beacon_url": "https://myserver.com/beaconRX2/play"
    }]
  }

방법 DELETE
URL https://api.bcovlive.io/v1/ssai/beaconsets/beaconset/beacon_set_id
머리글 X-API-KEY: your API KEY

샘플 응답

응답은 다음과 같습니다.

  {
    "beacon_set_id": "Beacon set ID",
    "deleted": true
  }
