    라이브 캐치업 API

    이 항목에서는 라이브 캐치업 API (타임시프트) 를 사용하여 캐치업으로 라이브 작업을 만드는 방법에 대해 설명합니다.

    서문

    Live Catch-up API는 라이브 스트림의 시작 또는 일부에 대한 재생 기능을 제공하는 Live API에 추가되어 늦은 시청자가 "추적"할 수 있도록합니다.

    캐치 업 기능은 새로운 기능을 통해 실시간 작업에 추가됩니다. timeshift라이브 작업을 만들 때 사용되는 개체 필드입니다. 세부 사항timeshift개체 필드는 아래 섹션에서 자세히 설명합니다.

    TimeShift 필드

    그만큼timeshift개체는 라이브 작업 요청 생성을위한 요청 본문의 최상위 개체입니다. 아래 표는이 개체에 대한 필드를 자세히 설명합니다.

    timeshift필드
    필드 설명
    startover_duration 시작 기간의 크기 (초), 최대 604800 (7 일).

    만약timeshift.startover_duration요청에 지정된 경우 각 재생 목록에 대해 타임 시프트 지원 재생 URL이 생성됩니다. 이 URL은 라이브 엣지 재생을 지원합니다 (DVR 크기는live_duration매개 변수).

    또한 매개 변수를 추가하여 startover_duration에 정의 된 기간 내 임의 부분의 재생을 지원합니다.

    • start=ts -재생 시작의 유닉스 epoch 타임 스탬프
    • end=ts -재생 종료의 유닉스 에포크 타임 스탬프
    live_duration 라이브 창 크기 (초) 최대 300. 기본값은 4 x 세그먼트 기간입니다.
    live_delay 라이브 출력을 지연하는 시간 (초)입니다. 최소 및 기본값은 5 초입니다.
    type
    • single-독립형 시간 이동 작업용 (기본값)
    • 1 차-1 차 근무일 용 (반드시 SEP 작업 )
    • 백업-백업 시간 이동 작업용
    primary_job_id 기본 시간 이동 작업 ID를 참조합니다 (유형이 백업 인 경우 필수).
    skip_live_output 일반 라이브 또는 DVR 재생 목록을 생성하지 마십시오 (기본값은 false).

    참고

    • timeshift.type이 "backup"인 경우 timeshift URL은 API 응답에 포함되지 않습니다. 기본 작업에 대해보고 된 타임 시프트 URL을 사용합니다.
    • 시간 이동 출력에 DRM이 필요한 경우drm평상시처럼 작업 요청의 필드. 이 속성은 시간 이동 출력을 보호하기 위해 기본 작업에 대해서만 지정되어야합니다. 그러나 보호되지 않은 재생을 방지하려면 모든 백업 작업에 대해 skip_live_output을 true로 설정하는 것이 좋습니다.
    • 클리핑은 지난 24 시간으로 제한되며 실시간 중복과 마찬가지로 선택한 작업의 입력 만 포함됩니다.

    직업 세부 정보 얻기

    작업 목록을 요청하든 특정 작업에 대한 세부 정보를 요청하든timeshift_url타임 시프트가 활성화 된 작업의 모든 재생 목록에 포함됩니다. 호출자는이 URL에 매개 변수를 추가하여 시작 및 중지 시간을 Unix Epoch 타임 스탬프로 지정할 수 있습니다. 예:

    https://bcovlive-a.akamaihd.net/8d64308aafe34d17adb98af69e9eac13/us-east-1/6021289116001/ts-cf98fa7b2ee4450e-5df3aae6cb724fe1a4a932b4964d3e17/playlist_ts.m3u8?start=1593457200&end=1593460800

    만약start과end매개 변수가 포함되지 않은 경우 라이브 에지에서 재생됩니다.

    재생 토큰 생성

    작업 소스 가져 오기 API에서 타임 시프트 URL을 반환 할 수 있도록 재생 토큰에 새 필드가 추가되었습니다. 현재 SSAI는 타임 시프트가 활성화 된 작업에 대해 지원되지 않으므로application_id속성을 지정하면 안됩니다.

    만약ts_start과ts_end지정되면 이전 섹션에서 설명한대로 쿼리 매개 변수로 재생 URL에 추가됩니다.

    새로운 분야

    새로운 재생 토큰 필드
    필드 설명
    ts 시간 이동 재생 URL 선택 (true | false, 선택 사항, 기본값은 false)
    ts_start 시작 창 시작을위한 Unix epoch 타임 스탬프 (숫자, 선택 사항)
    ts_end 시작 기간 종료에 대한 Unix epoch 타임 스탬프 (숫자, 선택 사항)
