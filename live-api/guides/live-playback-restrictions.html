소개

라이브 작업에 대해 재생 제한을 활성화하여 다음을 포함하여 DRM에서 제공하는 보호 기능을 확장 할 수 있습니다.

  • 특정 사용자 에이전트로 재생 제한
  • IP 주소 수 제한
  • 총 사용자 제한

포괄적 인 제한 사항 집합의 경우 Playback Rights API를 사용하여 구성 할 수 있습니다. 이 경우 를playback_rights_id비디오와 연결합니다. 자세한 내용은 개요의에셋 수준 제한섹션을 참조하십시오. Brightcove 재생 제한문서.

런타임 제한의 경우 JWT (JSON Web Token) 클레임을 지정할 수 있습니다. 이러한 제한에는 스트림 동시성 및 장치 제한이 포함됩니다. 자세한 내용은 개요의런타임 제한섹션을 참조하십시오. Brightcove 재생 제한문서.

라이브 스트림을 사용하면 사용자의 DRM CRT를 재정의 할 수도 있습니다. 자세한 내용은 개요의라이센스 키 보호섹션을 참조하십시오. Brightcove 재생 제한문서.

어떤 모드를 선택하든 Video Cloud 비디오가 생성되어 라이브 작업에 연결되도록 작업 요청에videocloud.video속성이 포함되어야 합니다. Video Cloud 비디오에서playback_rights_id속성을 업데이트하여 CMS API를 통해 재생 제한을 변경할 수있습니다.

당신은 또한 필요합니다 JSON 웹 토큰(JWT) 생성적절한 주장과 함께. JWT는 모든 라이센스 서버 요청에 포함되어야합니다. 자체 플레이어를 구현하는 경우 DRM 외에BCOV-Auth헤더 또는bcov_auth쿼리 파라미터에 이 값을 전달해야 합니다.토큰).

라이브 작업 생성

재생 제한을 활용하는 라이브 작업을 만들 수 있습니다. 라이브 스트림 생성에 대한 자세한 내용은 브라이트코브 라이브 API를 사용하여 라이브 스트림만들기문서를 참조하십시오.

새로운 분야

다음 새 필드는 라이브 작업에 재생 제한을 추가하는 데 사용됩니다.

  • drm객체:

    require_playback_token (부울)
    라이선스를 요청할 때 재생 인증 JWT가 필요합니다.
    check_playback_rights (부울)
    라이선스를 요청할 때 재생 인증 JWT가 필요합니다. 동영상 또는 JWT와 연결된 재생 권한 ID를 사용하여 제한을 적용합니다.

  • videocloud.video객체:

    이 필드는 CMS API를 사용하여 비디오에 추가해야 합니다. ~ 후에 비디오를 만들 때 추가할 수 없기 때문에 라이브 작업을 만듭니다.
    playback_rights_id (끈)
    지정된 재생 제한을 비디오와 연결합니다.

라이브 작업에 제한 추가

라이브 스트림을 만든 후 CMS API를 사용하여 라이브 작업에 대해 만든 비디오에 재생 제한을 추가할 수있습니다.

CMS API 요청

비디오에 재생 권한 ID를 추가하려면비디오 업데이트다음 요청 본문을 사용하여 CMS API에 요청합니다.

{
	"playback_rights_id": "your playback_rights_id"
}
