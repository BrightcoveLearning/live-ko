개요

프리미엄 또는 기밀 콘텐츠를 제공하는 경우 보안을 유지할 수있는 모든 방법을 모색하게됩니다. 최종 사용자가 적절한 권한없이 비디오 URL을 복사하거나 콘텐츠를 계속 재생하는 것을 원하지 않습니다. Brightcove는 RTMP, HLS 및 DASH URL 서명을 지원합니다.

자체 토큰을 생성하는 경우이 프로세스에 한 단계가 있습니다. 라이브 시스템이 토큰 화 된 URL을 생성하도록하려면 다음 섹션에서 자세히 설명하는 두 단계를 수행하여 라이브 스트림에 대한 토큰 인증을 활성화해야합니다.

현재 토큰 인증은 Akamai EdgeAuth 2.0에서만 지원됩니다.

이행

참고: 고유한 토큰을 만들어 URL에 추가하는 경우 1단계만 완료해야 합니다.

1 단계: 라이브 직업 만들기

토큰 인증을 활성화하려면 작업을생성할 때 요청 본문에 다음을 포함합니다.

  "add_cdns": [{
    "label": "My CDN",
    "prepend": "mycdn-a.akamaihd.net",
    "protocol": "https",
    "vendor": "akamai",
    "token_auth": {
      "auth_type": "Akamai2.0",
      "key": "AKAMAI_TOKEN_PASSWORD",
      "token_name": "AKAMAI_TOKEN_NAME",
      "media": {
        "start_time": "now",
        "end_time": 1495697388,
        "ttl": 60
      }
    }
  }]

관련 필드는 다음 표에 설명되어 있습니다.

라이브 작업을위한 TTL 필드
들 값
start_time now또는 유닉스/포믹스 시작 시간
end_time unix / posix 종료 시간
ttl 수명 (초)

2 단계: 라이브 작업이 생성 된 후

자체 토큰을생성하지 않고 Live API에서 토큰화된 URL을 생성하도록 하려는 경우에만 이 단계를 완료하십시오.

라이브 작업이 생성 된 후 마스터 매니페스트 URL을 생성하기 위해 다른 요청을해야합니다. 이렇게 하려면 다음 주소로POST요청을 보냅니다.

  https://api.bcovlive.io/v1/jobs/JOB_ID/authurls

요청 본문으로 다음 JSON을 포함합니다.

  {
    "ttl": number in seconds for time to live
  }

메모

  • 위의 두 단계의ttl값은 DVR 창보다 크거나 같아야 합니다. 그렇지 않으면 DVR 창이ttl값으로 제한됩니다.
  • http와 https가 모두 지원됩니다.
  • 은token_name 5-12자여야 하며 Akamai에 구성된 것과 일치해야 합니다.
  • start_time선택 사항이며"now" (기본값) 또는 에포크 시간 값 (초) 일 수 있습니다.
  • end_time선택 사항이며 에포크 시간 값 (초) 일 수 있습니다. end_timettl필수입니다. 지정되지 않은 경우, ttllive_dvr_sliding_window_duration과 중 더 큰 것으로 가정합니다. vod_sliding_window_duration
