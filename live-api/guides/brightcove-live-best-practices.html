브라이트코브 라이브: 모범 사례

이 항목에서는 Live API를 사용하여 실시간 스트림을 만드는 모범 사례에 대한 가이드를 제공합니다.

개요

브라이트코브 라이브는 라이브 스트리밍 이벤트 또는 연중무휴 라이브 스트림을 만들기 위한 강력한 서비스를 제공합니다. 이 가이드는 실시간 스트림을 최적화하기위한 권장 사항을 설명합니다.

입력 대역폭

고품질의 안정적인 입력 스트림을 제공하는 것이 시청자에게 최상의 사용자 경험을 보장하는 유일한 방법입니다. 좋은 입력 스트림은 한 위치에서 일관되게 사용할 수있는 가장 높은 대역폭에서 최상의 비디오 품질을 제공합니다.

  • 최소 입력 대역폭: 2.5mbps
  • 최대 입력 대역폭: 10mbps

지원되는 인코더

보다라이브 이벤트에 지원되는 인코더 Live와 함께 작동하는 것으로 알려진 인코더 목록. 다른 인코더도 작동 할 수 있지만 테스트되지는 않았습니다.

지원되는 CDN

  • Akamai
  • Amazon CloudFront
위의 CDN은 공식적으로 지원되지만 다른 파일 기반 CDN도 작동해야 합니다.

재시도

인코더에서 RTMP 연결 재 시도를 활성화하는 것이 좋습니다. 5 초의 재시도 간격으로 많은 재 시도를 시도하면 인코더와 진입 점 사이의 간헐적 인 연결 문제가 완화됩니다.

자주하는 질문

라이브 작업을 생성 한 후 얼마나 빨리 스트리밍을 시작해야합니까?
브라이트코브 라이브에는 상태가 에서waiting 로finishing 전환되는 두 가지 조건이 있습니다.
  1. 작업이대기상태 (아직 시작되지 않음) 이고max_waiting_time_ms이 경과한 경우 작업이 완료/비활성화됩니다.
  2. 작업이연결이 끊긴상태 (시작되었지만 연결이 끊긴 경우) reconnect_time가 경과된 경우 작업이 완료/비활성화됩니다.

event_length 이 30분 이상인 경우 30분 후에 작업이 종료됩니다. 이 30분event_length 미만이면 작업이 에서 종료됩니다event_length .

예를 들어event_length 가 60분이면 라이브 작업이 30분 안에 종료됩니다. event_length 이 15분이면 라이브 작업이 15분 내에 종료됩니다.

는 대기reconnect_time 상태에는 영향을 주지 않습니다.

동시 라이브 job_settings의 제한 사항은 무엇입니까?

최대 5개의 활성대기, 시작되지 않은작업은 언제든지 허용됩니다.

동시 작업에 대한 추가 제한 사항:

  • channel (24x7) 작업 수는 지역당 0 또는 낮은 숫자로 제한됩니다 (계정 유형에 따라 다름).
  • 동시실행 중인event작업의 수는 리전별로 제한되며 일반적으로 100으로 제한됩니다.
  • 동시 수연결 대기 중event작업은 5개로 제한됩니다.
  • 리전당 SEP 작업 수는 3개 또는 10개로 제한됩니다 ( 지원되는 AWS 리전 참조 ).

이러한 제한은 Support에서 계정 수준에서 조정할 수 있습니다. 추가 용량이 필요한 경우 계정 관리자에게 문의하십시오.
