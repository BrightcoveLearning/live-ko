PAS및 EPA는 현재 제한된 가용성 릴리즈입니다. 이러한 서비스를 사용하려면 계정 관리자에게 문의하십시오.

소개

DRM에서 제공하는 보호를 확장하기 위해 라이브 작업에 대해 PAS 또는 EPA를 활성화 할 수 있습니다. PAS는 제한된 제한 집합을 제공합니다. 특정 사용자 에이전트로 재생을 제한하거나 IP 주소 변경 또는 총 사용 횟수를 제한합니다. Live 사례에서는 고객이 해당 사용자의 DRM CRT를 재정의 할 수도 있습니다. 이러한 제한은 재생 권한 JWT에서 직접 지정됩니다 .

EPA는 훨씬 더 포괄적 인 제한을 제공합니다. 이러한 제한 사항 중 하나는 재생 권한 관리 API를 통해 구성되며 재생 권한을 사용하는 것으로 참조됩니다id . 이것id비디오와 연결할 수 있습니다( playback_rights_id아래에 표시된 속성) 또는 JWT . 두 번째 세트에는 동시성 제한 및 장치 수와 같은 사용자 수준 제한이 포함됩니다. 이는 JWT 클레임 (사용자 ID [v uid클레임] 을 포함해야 함) 에서 직접 지정됩니다.

두 모드의 경우 Video Cloud 비디오가 생성되어 라이브 작업에 연결되도록 작업 요청에videocloud.video속성이 포함되어야 합니다. Video Cloud 비디오에서playback_rights_id속성을 업데이트하여 CMS API를 통해 재생 제한을 변경할 수있습니다.

또한 사용 중인 항목에 따라 PAS 또는 EPA에 적합한 클레임이 포함된 JWT를생성해야 합니다. JWT는 모든 라이센스 서버 요청에 포함되어야합니다. 자체 플레이어를 구현하는 경우 DRM 외에BCOV-Auth헤더 또는bcov_auth쿼리 파라미터에 이 값을 전달해야 합니다.토큰).

EPA / PAS로 라이브 작업 생성

다음 새 필드는 라이브 작업에 EPA 및 / 또는 PAS를 추가하는 데 사용됩니다.

drm객체에 대한 새 필드:

require_playback_token (부울)
PAS를 활성화합니다. 라이선스를 요청할 때 재생 인증 JWT가 필요합니다.
check_playback_rights (부울)
EPA를 활성화합니다. 라이선스를 요청할 때 재생 인증 JWT가 필요합니다. 동영상 또는 JWT와 연결된 재생 권한 ID를 사용하여 제한을 적용합니다.

videocloud.video객체의 새 필드:

이 필드는 CMS API를 사용하여 비디오에 추가해야 합니다. ~ 후에 비디오를 만들 때 추가할 수 없기 때문에 라이브 작업을 만듭니다.
playback_rights_id (끈)
지정된 EPA 재생 권한을 비디오와 연결합니다.

	"playback_rights_id": "your playback_rights_id"
}
