    라이브와 함께 PAS/EPA 사용

    이 항목에서는 라이브 스트림과 함께 PAS (재생 권한 관리 서비스) 또는 EPA (재생 권한 관리 서비스) 를 사용하는 방법에 대해 설명합니다.
    우선권과 EPA현재 제한 가용성 릴리스에 있습니다. 이러한 서비스 사용에 관심이 있으면 계정 관리자에게 문의하십시오.

    서문

    DRM에서 제공하는 보호를 확장하기 위해 라이브 작업에 대해 PAS 또는 EPA를 활성화 할 수 있습니다. PAS는 제한된 제한 집합을 제공합니다. 특정 사용자 에이전트로 재생을 제한하거나 IP 주소 변경 또는 총 사용 횟수를 제한합니다. Live 사례에서는 고객이 해당 사용자의 DRM CRT를 재정의 할 수도 있습니다. 이러한 제한은 재생 권한에 직접 지정됩니다. JWT .

    EPA는 훨씬 더 포괄적 인 제한을 제공합니다. 이러한 제한 사항 중 하나는 재생 권한 관리 API를 통해 구성되며 재생 권한을 사용하여 참조됩니다. id . 이id비디오와 연결될 수 있습니다 ( playback_rights_id아래 표시된 속성) 또는 JWT . 두 번째 세트에는 동시성 제한 및 장치 수와 같은 사용자 수준 제한이 포함됩니다. 이는 JWT 클레임에 직접 지정됩니다 (사용자 ID [v uid청구]).

    어느 모드에서든 작업 요청에는videocloud.video속성을 사용하여 Video Cloud 비디오가 생성되고 라이브 작업에 연결됩니다. 그만큼playback_rights_id Video Cloud 비디오에서 속성을 업데이트하여 재생 제한을 변경할 수 있습니다. CMS API .

    또한 다음을 수행해야합니다. JWT 생성사용하는 것에 따라 PAS 또는 EPA에 적절한 클레임. JWT는 모든 라이센스 서버 요청에 포함되어야합니다. 자신의 플레이어를 구현하는 경우이 값을BCOV-Auth헤더 또는bcov_auth쿼리 매개 변수 (DRM 토큰에 추가).

    EPA / PAS로 라이브 작업 생성

    다음 새 필드는 라이브 작업에 EPA 및 / 또는 PAS를 추가하는 데 사용됩니다.

    새로운 필드drm목적:

    require_playback_token (부울)
    PAS를 활성화합니다. 라이센스 요청 시 재생 인증 JWT가 필요합니다.
    check_playback_rights (부울)
    EPA를 활성화합니다. 라이센스 요청 시 재생 인증 JWT가 필요합니다. 동영상 또는 JWT와 연결된 재생 권한 ID를 사용하여 제한을 적용합니다.

    에 대한 새 필드videocloud.video목적:

    이 필드는 CMS API를 사용하여 비디오에 추가해야합니다. 후비디오가 생성 될 때 추가 할 수 없기 때문에 라이브 작업을 생성합니다.
    playback_rights_id (끈)
    지정된 EPA 재생 권한을 비디오와 연결합니다.
