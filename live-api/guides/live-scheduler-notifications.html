소개

작업을 완료한 후 스케줄러는 http 메소드 POST를 사용하여 구성된 엔드포인트에 어떤 작업이 수행되었는지 및 성공/실패에 대한 세부 정보와함께 알림을 보냅니다.

예정된 작업에 대한 사전 알림을 구성 할 수도 있습니다. 참조라이브 API 참조알림 URL 및 설정 지정에 대한 자세한 내용과 SEP 작업의 시작/중지 시간 예약 및 클립 예약의 차이점을 확인하십시오.

스케줄러 시작 / 중지
  • 알림 URL은 요청 본문 개체의 최상위 수준에서 정의됩니다.
  • 알림은활성화및비활성화객체에서 예약됩니다 (알림을 보내기 위한 이벤트 전 시간 (초).

예

{
	"job_id": "your Job ID",
	"activation": {
		"time_utc": 1612274100,
		"notification": 600
	},
	"deactivation": {
		"time_utc": 1612281600,
		"notification": 600
	},
	"description": "My Tuesday broadcast",
	"notification_url": "http://example.com/live/jobstartstop/callbacks"
}
클립 스케줄러
  • 알림 URL은 요청 본문 개체의 최상위 수준에서 정의됩니다.
  • 알림은클립객체에서 예약됩니다 (작업이 알림 전송을 시작하기까지의 시간 (초).

예

{ "job_id": "작업 ID", "설명": “내 예약된 클립”, “알림_URL”: "http://example.com/live/clip/callbacks" , “클립”: { "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "name": "My Scheduled Clip" } }}, “타임_utc”: 1612879800, “알림”: 600 }}

알림 본문

서버가 notification ation_url에 알림을보내면 요청에 JSON 본문이 포함됩니다. 필드는 아래 표에 자세히 설명되어 있습니다.

알림 본문 필드
동작 액션 이름. 동작 보기
워크 플로우 작업 완료 후 업데이트 된 워크 플로. 구조는 워크 플로 유형에 따라 다릅니다.
오류 작업을 완료하는 동안 오류가 발생하면이 필드에 오류가 포함됩니다. 값은 작업 유형에 따라 다릅니다.

행위

다음 표에는 워크 플로 유형별 작업이 자세히 설명되어 있습니다.

행위
이름 워크 플로우 유형 기술
notify-activate jobstartstop 작업이 곧 활성화 될 것이라는 알림을받습니다.
activate jobstartstop 작업 활성화가 완료되었습니다. “오류”가있으면 활성화에 실패하고 비활성화가 취소됩니다.
notify-deactivate jobstartstop 작업이 곧 활성화 될 것이라는 알림을받습니다.
deactivate jobstartstop 작업 비활성화가 완료되었습니다. “error”가 있으면 해제에 실패했습니다.
notify-clip clip 클립이 곧 요청 될 것이라는 알림을받습니다.
clip clip 클립 요청이 완료되었습니다. “error”가있으면 요청이 실패했습니다.
