    라이브 API를 통한 라이브 DRM

    이 항목에서는 LIve API를 통한 라이브 스트림 용 DRM 추가에 대해 설명합니다.

    서문

    DRM은 다음을 포함하여 라이브 작업에 추가 할 수 있습니다. DRM작업을 만들 때 요청 본문의 개체입니다. 그만큼DRM객체에는 두 가지 속성이 있습니다.

    • modes배열 또는 "all", "fairplay", "playready", "widevine", "widevine : dash"또는 "widevine : hls"중 하나 이상으로 설정
    • token_expires_in 43200 (기본값)에서 31622400 (토큰이 만료되기까지의 시간 (초)) 사이의 숫자로 설정

    환경mode모든 사용 가능한 DRM 유형이 출력에 적용됩니다. 기본적으로 DRM은 다음과 같이 출력에 적용됩니다.

    • 페어플레이: HLS 출력에 적용
    • 재생 준비: DASH 출력에 적용됨
    • Widevine : HLS 출력에 적용

    Widevine은 DASH에 적용 할 수 있습니다.   HLS 대신 출력. DASH를 사용하여 Widevine을 기본값으로 설정하려면 계정 수준 재정의가 필요합니다. 이 재정의가 계정에 적용되도록 하려면 지원 서비스에 문의하십시오. 다음 섹션에서는 지정하는 방법을 볼 수 있습니다. widevine:hls또는widevine:dash계정의 기본값에 관계없이 특정 작업에 대해. 계정/작업 구성 방법에 따라 API 응답 (아래 표시된 것과 같은) 에는widevine:hls또는widevine:dash출력이 포함되지만 둘 다 포함되지는 않습니다.

    Microsoft와의 서비스 계약에 정의된 PlayReady에 대한 동시성 제한이 있습니다. 동시성 관리에 대한 자세한 내용은이 Microsoft 문서를참조하십시오. 비디오에 대한 동시성 제한을 초과하면 최대 수를 초과하는 시청자에 대한 재생이 실패합니다.
