라이브 API를 통한 라이브 DRM

이 항목에서는 LIve API를 통한 라이브 스트림 용 DRM 추가에 대해 설명합니다.

소개

작업을 만들 때 요청 본문에DRM객체를 포함시켜 DRM을 라이브 작업에 추가할 수 있습니다. DRM객체에는 두 가지 속성이 있습니다.

  • modes배열 또는 “모두”, “페어플레이”, “플레이레디”, “와이드 빈”, “와이드 빈:대시” 또는 “widevine:hls” 중 하나 이상으로 설정합니다.
  • token_expires_in 43200 (기본값)에서 31622400 (토큰이 만료되기까지의 시간 (초)) 사이의 숫자로 설정

mode all로 설정하면 사용 가능한 모든 DRM 유형이 출력에 적용됩니다. 기본적으로 DRM은 다음과 같이 출력에 적용됩니다.

  • 페어플레이: HLS 출력에 적용
  • 플레이레디: 대시 출력에 적용
  • 와이드바인: HLS 출력에 적용

Widevine은 HLS 대신 DASH  출력에 적용할 수 있습니다. DASH로 Widevine을 만들려면 기본값으로 계정 수준 재정의가 필요합니다. 이 재정의를 계정에 적용하려면 지원팀에 문의하세요. 다음 섹션에서는 계정의 기본값에 관계없이 특정 작업을widevine:hls지정하거나widevine:dash지정할 수 있는 방법을 확인할 수 있습니다. 계정/작업 구성 방법에 따라 API 응답 (아래 표시된 것과 같은) 에는widevine:hls또는widevine:dash출력이 포함되지만둘 다 아닙니다.

Microsoft와의 서비스 계약에 정의된 PlayReady에 대한 동시성 제한이 있습니다. 보다이 마이크로소프트 문서동시성 관리에 대한 정보는 동영상의 동시성 한도를 초과하면 최대 수를 초과하는 시청자가 재생되지 않습니다.
