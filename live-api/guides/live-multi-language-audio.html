요구 사항

  • 다국어 오디오 기능이 활성화 된 Live API 용 계정 설정 (자세한 내용은 계정 관리자에게 문의)
  • Live API 용 API 키 (계정 설정시 제공됨)
  • RTP 프로토콜을 지원하는 인코더. 다국어 오디오 지원은rtp또는rtp-fec프로토콜로 제한됩니다.
  • 작업을 만들 때 각 오디오 트랙의 PID를 알아야합니다.

라이브 작업 생성

작업에 여러 오디오 트랙을 추가하려면 라이브 작업을 만들 때 요청 본문에 오디오 트랙 목록을 제공해야합니다.

{
  "live_stream": true,
  "region": "us-west-2",
  "protocol": "rtp",
  "cidr_whitelist": [ /* omitted… */ ],
  "outputs": [ /* omitted… */ ],
  "alternate_audio": {
    "tracks": [ { Track Details } ]
  }
}

Track객체에 대해 지원되는 필드

아래 표에는track객체에서 지원되는 필드에 대한 전체 설명이 나와 있습니다.

트랙 필드
들 기술
pid 필수입니다. Wowza로 전송 된 스트림 파일에 지정된 트랙의 수신 TS에서 선택할 PID입니다.
language 필수입니다. 사용할 언어의 코드입니다. 현재 이것은 유연하며 사양에 따라 요구 사항/지원되는 다양한 언어 형식, 특히 요구 사항에 따라 ISO-639 을 다루는 RFC5456/BCP47으로 제공 될 수 있습니다. https://tools.ietf.org/html/rfc5646

HLS > LANGUAGE필드

대시 > AdaptationSet:lang필드
variant 선택 사항입니다 . DASH 역할 스키마 값 (메인, 대체, 설명, 보충, 더빙)에 매핑됩니다. 대시 역할에 사용됩니다urn:mpeg:dash:role:2011 . 기본값은main입니다.

HLS > 특성 참조 https://tools.ietf.org/html/rfc8216#section-4.3.4.1
label 선택 사항입니다 . 가능한 경우 플레이어가 사용하는 것이 선호되는 트랙에 사용할 텍스트 설명입니다. 사용 위치: HLS > NAME필드. 기본값은 각 트랙의language ID와 함께 입니다 (예: 지정되지 않은en-0경우).
streams [] 선택 사항입니다 . 복사가 없으면 기본 오디오 설정 만 사용합니다.

HLS > GROUP-ID . (목적)

예: { “source”: “720p” } -출력 소스의 일치 레이블.

재생 목록 출력 유형의streams필드를 모방합니다.
default 기본값을 제외하고 필수항목입니다. 어떤 오디오 트랙이 비디오DEFAULT스트림에 머싱될EXT-X-MEDIA것뿐만 아니라 어떤 오디오 트랙이 표시되어야 하는지*

없는 경우 첫 번째가 기본값입니다.

* 여러 재생 목록이 다른 기본값으로 정의 된 경우 최상위 기본 트랙은 다중 재생 목록입니다.

인코더 구성

RTP를 통해 MPEG-TS를 생성 응답에서 stream_url로 전달하도록 인코더를 구성합니다. 요청 본문에 제공된 오디오 트랙이 인코더에서 올바른 PID로 구성되어 있는지 확인합니다.

재생 테스트

재생을 확인하려면 playback_url을브라이트코브 플레이어또는 Safari에 로드합니다.

DRM_Playback

현재 DRM은 Catch-up (timeshift) 서비스를 통해서만 지원됩니다. 작업 만들기 요청 본문에 다음 필드를 추가합니다.

{
  "drm": {
    "modes": [
      "all"
    ]
  },
  "timeshift": {
    "type": "single",
    "startover_duration": 36000,
    "skip_live_output": true
  }
}

응답에는timeshift_url재생에 사용할 DRM 출력과 a가 포함됩니다.
