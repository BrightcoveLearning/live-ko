일반 정보

RTMP 출력을 사용하려면 다음 사항에 유의하십시오.

  • 이벤트 시간또는 디지털 마케팅 패키지를 구매해야 합니다.
  • RTMP 출력 시간은 이벤트 시간에 대해 청구됩니다.
  • 라이브 작업 당 5 개의 출력으로 제한됩니다.
  • 레이블이 스트림에 대해 선택한 출력 변환 중 하나를 선택할 수label있는 위치를 정의할 수 있습니다. 가 설정되지 않은 경우label출력은 기본적으로 입력 소스로 설정됩니다.
  • 표준 작업 및 반복 이벤트 (SEP)와 함께 사용할 수 있습니다.
  • 이벤트 또는 24/7 스트림에 사용할 수 있습니다.
  • 라이브 스트림과는 별도로 시작 및 중지 할 수 있습니다.
  • RTMP 출력 기능이 구현되기 전에 생성 된 SEP 작업의 경우 기능이 완전히 작동하지 않습니다. API에서는 RTMP 출력이 실제로 생성 되더라도 실패가 표시됩니다. 최상의 결과를 얻으려면 RTMP 출력 기능을 사용하는 새 SEP 작업을 만드십시오.
  • 표준 작업 (비 SEP)과 함께 RTMP 출력을 사용하는 경우 작업이 종료되면 RTMP 출력 스트림이 종료됩니다.
  • RTMP 출력 ~하다지원하다rtmps url이며 소셜 미디어 플랫폼에서 지원하는 경우 사용하는 것이 좋습니다. 또한 2019년 5월 1일부터 페이스북은 RTMPS만 허용합니다.
  • RTMP URL에 대해 지원되는 현재 형식은유일한형식입니다rtmps://host:port/application/streamName .

알림

rtmp_output_state_changed이벤트에 대한 알림을 설정할 수 있습니다. 다음 상태에 대한 알림이 전송됩니다.

비
RTMP 출력 상태
RTMP 출력 상태 ( rtmp_output_state_changed알림에 대해 다음과 같이state보고됨)
starting 스트림이 시작됩니다.
connected 인코더가 연결되었습니다.
Disconnected 인코더 연결이 끊어졌습니다.
error 스트림 연결에 실패했습니다.
pending_deactivation 스트리밍 작업자의 정리를 위해 진입 점이 대기열에 추가되었습니다.
deactivation_in_progress 진입 점이 스트리밍 작업자와 연결이 끊어졌습니다.
cancelled 진입 점이 취소되었습니다.
finished 진입 점이 성공적으로 완료되었습니다.

RTMP 출력 생성

먼저 평소와 같이 라이브 작업을 생성 한 다음 RTMP 엔드 포인트를 생성하기위한 추가 요청을해야합니다.

참고: 작업 활성화/비활성화와 같은 특정 상황에서이 API 호출이 오류를 반환하는 경우가 있습니다. 일시 중지 후 다시 시도하십시오.

끝점

	https://api.bcovlive.io/v1/jobs/{jobId}/rtmpouts

방법

POST

요청 본문

	{
		"url": "rtmps://host:port/application/streamName",
		"output_label": "480p"
	}
본문 필드 요청
분야 명 필수 유형 기술
url 예 끈 RTMP 출력에 대한 URL - 예제rtmp://live-api-s.facebook.com:80/rtmp/my_stream
output_label 아니 끈 RTMP 출력의 소스로 사용할 작업 출력의 레이블입니다. 대신 RTMP 입력을 사용하려면이 매개 변수를 생략하십시오.

샘플 응답

	{
		"host": "live-api-s.facebook.com",
		"port": 80,
		"application": "rtmp",
		"streamName": "my_stream",
		"rtmp_out_id": "asdfg-lkjh"
	}
