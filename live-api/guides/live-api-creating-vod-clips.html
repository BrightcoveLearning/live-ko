라이브 API: VOD 클립 만들기

이 항목에서는 라이브 스트림에서 주문형 비디오 (VOD) 클립을 만드는 방법을 보여줍니다.

개요

클립은 라이브 스트림에서 추출한 비디오입니다. S3 버킷, FTP 사이트 또는비디오 클라우드계정으로 전송할 수 있습니다. 클립은 MP4 비디오로 생성되며 모든 경우에 대상으로 전송됩니다. Video Cloud의 경우 , MP4는 인제스트 시스템에 의해 트랜스코딩되며, 비디오에 대해 어떤 종류의 변환이 생성되는지는 사용된 수집 프로필에 따라 달라집니다.

클립에 대한 정의는/vods엔드포인트를 사용하여 생성됩니다.

클립은 여러 가지 방법으로 만들 수 있습니다.

  • 라이브 스트림 이벤트에 대한 SMPTE 타임코드와 함께stream_start_timecode및/또는stream_end_timecode정의되면 인코더가 필요합니다. 타임코드 정보 전송
  • 시작 시간을 기준으로start_time및/또는end_timestream_start_time정의됨 ()전체 라이브 스트림 이벤트
  • Epoch (유닉스) 시간 (초) 으로start_time및/또는end_time정의됨
  • 함께duration
  • VOD API는암호화된 작업 또는 DRM 보호작업에 사용할수 있습니다. 현재 Live Module은이를 지원하지 않지만 향후 릴리스에서 지원할 예정입니다.

메모

  1. 클립을 가능한 한 빨리 사용할 수 있도록 먼저 세그먼트에 정확한 클립을 만든 다음 사용할 수있는 즉시 프레임에 정확한 클립으로 교체합니다.
  2. duration지정하면 결과 클립은 다음과 같습니다.
    • 작업이 활성 상태이고 아직 활성 상태인 경우: (요청 시간 - 기간) 에서 (요청 시간)
    • 작업이 완료된 경우 ( finished_at - 기간) 에서 ( finished_at )
  3. start_time그리고 다음 두 가지를 모두 지정하는 경우end_time :
    • 작업이 활성 상태이고 여전히 활성 상태인 경우: Epoch 시간 창이 완전히created_at내리고 요청 시간이 있는 한 클립이 만들어집니다.
    • 작업이 완료된 경우: Epoch 시간 창이 완전히created_at내리고finished_at있는 한 클립이 만들어집니다.
  4. SSAI를 사용하는 라이브 스트림 클립에는 광고가 포함되지않습니다.
  5. 클립은 이벤트 후 최대 7 일까지 생성 할 수 있습니다. SEP의 경우 다음 활성화 또는 7일 (둘 중 짧은) 까지 만들 수 있습니다.
  6. VOD API는 스트림에있는 것 이외의 콘텐츠를 추가하지 않습니다. 300 초 길이의 라이브 스트림에 350을 지정하면 출력 길이는 300 초가됩니다.
  7. 라이브 스트림은 방송되는 그대로 저장되고 이벤트 종료 후 7 일 동안 즉시 사용할 수 있으므로 클리핑을 위해 DVR 지원 라이브 스트림을 사용할 필요가 없습니다.
  8. Brightcove Live 클리핑은 최고 해상도 출력과 동일한 해상도의 클립 만 생성합니다. 소스 입력 해상도와 일치하지 않습니다 (최고 해상도 출력과 동일하지 않은 경우).

클립을 여러 대상으로 보낼 수도 있습니다.

  • 비디오 클라우드계정
  • FTP 서버
  • S3 버킷

클립을 지정하면 출력 ~해야하다포함하다어느 하나 NS url목적지또는 NS videocloud비디오 생성 및 클립 수집을 자세히 설명하는 개체비디오 클라우드 .

참고: 라이브 스트림이 실행되는 동안 클립을 만들수있습니다. 이렇게 하려면 클립의 시작 시간과 종료 시간을 Epoch 시간 또는 라이브 스트림의시작시간을 기준으로 정의해야 합니다.

신임장

클립을 보내는 대상에 액세스하는 데 자격 증명이 필요한 경우 Live API의 자격 증명 작업을 사용하여 이러한 자격 증명을 만들 수 있습니다. 보다 Live API에 대한 자격 증명 관리자세한 사항은.

라이브 계정과 연결된 Video Cloud 계정으로 클립을 보내는 경우 클리핑 작업에 자격 증명을포함해서는 안 됩니다.

요청 본문 - 비디오 클라우드

참고: 클립을 트랜스코딩하는 데 사용해야 하는 인제스트 프로파일을지정하지 말고 대신 계정 기본 프로필을 사용하는 것이 좋습니다. 이렇게 하면Video Cloud에서 클립을 만들 때 오류가 발생할 가능성이 줄어듭니다.

예 1: 스트림 시작에 상대적인 시작/종료 시간

요청 본문에는 시작 및 종료 시간과 클립을 보낼 위치에 대한 세부 정보가 포함됩니다. 다음은 스트림의 3분의 클립을 만들어 Video Cloud계정으로 전송하는 샘플 요청 본문입니다.

{
  "live_job_id":"PUT-LIVE-JOB-ID-HERE",
  "outputs":[
    {
      "label": "60 secs by stream from min 2 to min 3",
      "stream_start_time": 120,
      "stream_end_time": 180,
      "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL",
      "videocloud": {
        "video": {
        "name": "One Minute Clip",
        "tags": ["live", "clip"]
        },
          "ingest": {
            "capture-images": true
        }
      }
    }
  ]
}

이 예에서는 1분 길이의 클립을 만들어 Video Cloud로전송합니다. 클립 이름과 몇 개의 태그를 지정하지 않고인제스트 프로필계정 기본값이 사용되도록 재변환하고비디오 클라우드트랜스코딩 중에 클립에서 썸네일 및 포스터 이미지를 캡처합니다.

예 2: Epoch 시간의 시작/종료 시간

요청 본문에는 Epoch 시간의 시작 및 종료 시간과 클립을 보낼 위치에 대한 세부 정보가 포함됩니다. 다음은 스트림의 3분의 클립을 만들어 Video Cloud계정으로 전송하는 샘플 요청 본문입니다.

          "video": {

          },
            "ingest": {
            "capture-images": true

  ]
}

이 예에서는 특정 에포크 시간 (이 경우 2018년 1월 22일 8:24:54 GMT) 에 1분 길이의 클립을 만듭니다.

예 3: 스트림 시작에 상대적인 시작 시간이 있는 지속 시간

요청 본문에는 기간 및 stream_start_time, 클립을 보낼 위치에 대한 세부 정보가 포함됩니다. 다음은 스트림의 3분의 클립을 만들어 Video Cloud계정으로 전송하는 샘플 요청 본문입니다.

{
  "live_job_id":"PUT-LIVE-JOB-ID-HERE",

이 예제에서는 라이브 스트림이 시작된 후 5분 후부터 1분까지의 클립이 생성됩니다.

예 4: 시작 또는 종료 시간이 없는 지속 시간

요청 본문에는 Epoch 시간의 시작 및 종료 시간과 클립을 보낼 위치에 대한 세부 정보가 포함됩니다. 다음은 스트림의 3분의 클립을 만들어 Video Cloud계정으로 전송하는 샘플 요청 본문입니다.

예제 5: stream_start_timecode및 사용stream_end_timecode

요청 본문에는 HH:MM:SS:FF 시간 코드의 시작 및 종료 시간/프레임과 클립을 보낼 위치에 대한 세부 정보가 포함됩니다. 타임 코드를 사용하려면 인코더가 타임 코드를 전송해야합니다. 다음은 스트림의 50분의 클립을 만들어 Video Cloud계정으로 전송하는 샘플 요청 본문입니다.

요청 본문-S3

요청 본문에는 시작 및 종료 시간과 클립을 보낼 위치에 대한 세부 정보가 포함됩니다. 다음은 스트림의 3분의 클립을 만들어 S3 버킷으로 전송하는 샘플 요청 본문입니다.

{
  "live_job_id":"{{job_id}}",
  "outputs":[
    {
      "label": "last_30",
      "duration": 30,
      "url": "s3://YOUR_BUCKET_NAME/file_name.mp4",
      "credentials": "s3-credentials",
      "notifications": ["http://myserver.com/api/notification_listener?type=jvod"]
    }
  ],
}

이 예에서는 30 초 길이의 클립을 생성하여 S3 버킷으로 보냅니다. 클립의 파일 이름이 포함 된 버킷 URL과 저장된 S3 버킷 자격 증명의 이름 인 문자열을 제공합니다. 자격 증명은 Brightcove 지원에서 계정에 대해 설정할 수 있습니다.

schedule 속성-값 쌍의 맵 비디오 가용성에 대한 시작 및 종료 날짜-시간지도
